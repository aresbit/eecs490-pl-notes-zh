## 3.1 Lambda 演算 (Lambda Calculus)

Lambda 演算是计算理论的基石。它只有三种语法形式，却能表达任何可计算函数。理解 lambda 演算是理解函数式编程、类型系统乃至程序语义的前提。

### 3.1.1 语法 (Grammar)

Lambda 演算的 BNF 语法极为简洁：

```
e ::= x          变量 (Variable)
    | λx. e      抽象 (Abstraction)
    | e₁ e₂      应用 (Application)
```

其中 $$x$$ 来源于一个可数无穷的变量名集合 $$\mathcal{V} = \{a, b, c, \dots, x, y, z, \dots\}$$。

**变量 (Variable)** 是最基本的表达式，它只是一个名字引用。

**抽象 (Abstraction)** 定义了一个单参数匿名函数：$$\lambda x. e$$ 表示以 $$x$$ 为形参、$$e$$ 为函数体的函数。参数 $$x$$ 在函数体 $$e$$ 中被**绑定**。

**应用 (Application)** 表示函数调用：$$e_1 e_2$$ 将 $$e_1$$ 作为函数、$$e_2$$ 作为实参进行调用。应用是左结合的：

$$e_1 e_2 e_3 = (e_1 e_2) e_3$$

而抽象体向右延伸到最大可能范围：

$$\lambda x. \lambda y. e = \lambda x. (\lambda y. e)$$

**多参数函数**：lambda 演算中每个抽象只接受一个参数。多参数函数通过**柯里化 (Currying)**实现：

$$\lambda x. \lambda y. x + y$$

等价于一个接受 $$x$$ 后返回一个接受 $$y$$ 的函数。因此 $$\lambda x. \lambda y. e$$ 可简写为 $$\lambda x y. e$$。

### 3.1.2 变量作用域：自由变量与绑定变量 (Free and Bound Variables)

一个变量在表达式中要么是**自由的**要么是**绑定的**。

定义函数 $$FV(e)$$ 返回表达式 $$e$$ 中的自由变量集合，以及 $$BV(e)$$ 返回绑定变量集合：

**$$\mathbf{FV}(x) = \{x\}$$**：单独的变量是自由的。

**$$\mathbf{FV}(\lambda x. e) = \mathbf{FV}(e) \setminus \{x\}$$**：抽象 $$\lambda x$$ 绑定了 $$x$$，将其从自由变量集合中移除。

**$$\mathbf{FV}(e_1 e_2) = \mathbf{FV}(e_1) \cup \mathbf{FV}(e_2)$$**：应用中的自由变量是两个子表达式自由变量的并集。

**示例推导**：

对于表达式 $$\lambda x. (\lambda y. x y) z$$：

```
1. FV(z) = {z}
2. FV(x y) = FV(x) ∪ FV(y) = {x} ∪ {y} = {x, y}
3. FV(λy. x y) = FV(x y) \ {y} = {x, y} \ {y} = {x}
4. FV((λy. x y) z) = FV(λy. x y) ∪ FV(z) = {x} ∪ {z} = {x, z}
5. FV(λx. (λy. x y) z) = FV((λy. x y) z) \ {x} = {x, z} \ {x} = {z}
```

因此 $$z$$ 是唯一的自由变量，$$x$$ 和 $$y$$ 都是绑定的。一个没有自由变量的表达式称为**闭合表达式 (closed expression)** 或**组合子 (combinator)**。

### 3.1.3 α-变换 (α-conversion)

α-变换是对绑定变量进行重命名的操作，两个表达式仅绑定变量名不同时被视为 α-等价的。

**形式定义**：

$$\lambda x. e \to_\alpha \lambda y. (e[x \mapsto y])$$

其中 $$y$$ 是**新鲜的**（即 $$y$$ 不在 $$e$$ 的自由变量中出现），且 $$e[x \mapsto y]$$ 表示将 $$e$$ 中所有自由的 $$x$$ 替换为 $$y$$。

**α-等价的直观理解**：

$$\lambda x. x$$、$$\lambda y. y$$、$$\lambda z. z$$ 都表示恒等函数 (identity function)。它们只在一个称为"德布鲁因索引 (De Bruijn index)"的无名表示层面才统一。

**α-变换示例**：

```
1. λx. λy. x y
2. →α λa. λy. a y      [x 重命名为 a]
3. →α λa. λb. a b      [y 重命名为 b]
```

在整个 lambda 演算中，我们将 α-等价的表达式视为**相同的**表达式。这一约定使得替换的定义变得简洁——我们可以始终假设绑定变量已重命名为不与任何自由变量冲突的名字。

### 3.1.4 替换 (Substitution)

替换 $$e_1[x \mapsto e_2]$$ 是 lambda 演算中最关键的元操作。它的定义必须小心处理变量捕获问题。

**替换的归纳定义**：

1. $$x[x \mapsto t] = t$$
2. $$y[x \mapsto t] = y$$ 当 $$y \neq x$$
3. $$(e_1 e_2)[x \mapsto t] = (e_1[x \mapsto t])(e_2[x \mapsto t])$$
4. $$(\lambda x. e)[x \mapsto t] = \lambda x. e$$（$$x$$ 被重绑定，不再替换函数体中的自由 $$x$$）
5. $$(\lambda y. e)[x \mapsto t] = \lambda y. (e[x \mapsto t])$$ 当 $$y \neq x$$ 且 $$y \notin \mathbf{FV}(t)$$
6. $$(\lambda y. e)[x \mapsto t] = \lambda z. ((e[y \mapsto z])[x \mapsto t])$$ 当 $$y \neq x$$ 且 $$y \in \mathbf{FV}(t)$$，其中 $$z$$ 是新鲜的

情形 6 是**变量捕获避免 (capture avoidance)**：如果 $$t$$ 中包含自由变量 $$y$$，我们不能简单地将 $$x$$ 替换为含 $$y$$ 的 $$t$$ 在 $$\lambda y. e$$ 内部进行，否则自由出现的 $$y$$（来自 $$t$$）会被绑定。解决方案是将函数体中的绑定变量 $$y$$ 先 α-重命名为新鲜变量 $$z$$。

**替换示例**：

计算 $$(\lambda x. x y)[x \mapsto (\lambda z. z)]$$：

```
(λx. x y)[x ↦ (λz. z)]
= λx. x y           [情形 4: x 在 λx 处被重绑定]
```

计算 $$(\lambda x. y)[y \mapsto x]$$：

```
(λx. y)[y ↦ x]
= λx'. (y[x ↦ x'])  [情形 6: x ∈ FV(x), y ≠ x, 先将 x α-重命名为 x']
[此处我们处理 y ↦ x, 但 x 在 FV(x) 中，实际上:
 绑定变量是 x = λx, 替入项是 x (变量), FV(替换项) = {x}
 y ≠ x (这里被替换的是 y, 绑定量是 x), 且 x ∈ FV(x).
 不对, 我们替换的是 y, 替入项是 x. FV(x) = {x}.
 绑定量是 λx 中的 x. x ∈ FV(替换项) 成立.
 所以需要 α-重命名.]
```

更仔细地推导：

```
(λx. y)[y ↦ x]

这里: e = y (函数体), 替换 [y ↦ x], 抽象是 λx.

绑变量 y_var = x (来自 λx)
被替换变量 x_var = y
替入项 t = x

条件: y_var ≠ x_var (即 x ≠ y) ✓
      y_var ∈ FV(t) (即 x ∈ FV(x) = {x}) ✓

所以触发情形 6: 需要将 λx. y 中的 x α-重命名为 z (新鲜):
(λx. y) →α (λz. y)
然后: (λz. y)[y ↦ x] = λz. (y[y ↦ x]) = λz. x
```

结果：$$(\lambda x. y)[y \mapsto x] = \lambda z. x$$

### 3.1.5 β-归约 (β-reduction)

β-归约是 lambda 演算中唯一的计算规则。它描述的是**函数应用**的行为。

**β-归约规则**：

$$(\lambda x. e_1) e_2 \to_\beta e_1[x \mapsto e_2]$$

将一个抽象应用到一个参数上，就是将函数体中的形参替换为实参。

**β-归约的上下文规则**（它们规定了在哪里可以进行归约）：

- 如果 $$e_1 \to_\beta e_1'$$，则 $$e_1 e_2 \to_\beta e_1' e_2$$
- 如果 $$e_2 \to_\beta e_2'$$，则 $$e_1 e_2 \to_\beta e_1 e_2'$$
- 如果 $$e \to_\beta e'$$，则 $$\lambda x. e \to_\beta \lambda x. e'$$

**简单 β-归约示例**：

```
(λx. x) (λy. y y)
→β x[x ↦ (λy. y y)]
= (λy. y y)
```

**多个应用的归约**：

```
(λx. λy. x y) a b

首先，确认左结合:
= ((λx. λy. x y) a) b

内层应用:
(λx. λy. x y) a
→β (λy. x y)[x ↦ a]
= λy. a y

继续:
(λy. a y) b
→β (a y)[y ↦ b]
= a b
```

因此整个表达式归约到 $$a b$$。

**恒等函数的组合**：

```
(λf. λx. f (f x)) (λy. y) z

确认左结合:
= (((λf. λx. f (f x)) (λy. y)) z)

步骤1: (λf. λx. f (f x)) (λy. y)
→β (λx. f (f x))[f ↦ (λy. y)]
= λx. (λy. y) ((λy. y) x)

步骤2: (λx. (λy. y) ((λy. y) x)) z
→β ((λy. y) ((λy. y) x))[x ↦ z]
= (λy. y) ((λy. y) z)

步骤3: (λy. y) ((λy. y) z)
→β y[y ↦ ((λy. y) z)]
= (λy. y) z

步骤4: (λy. y) z
→β y[y ↦ z]
= z
```

### 3.1.6 正规形式与终止性 (Normal Form and Termination)

一个表达式如果没有可以继续 β-归约的子项，就处于**β-正规形式 (β-normal form)**。换句话说，表达式中不含任何形如 $$(\lambda x. e_1) e_2$$ 的子项（这称为 **β-可归约式 (β-redex)**）。

**定义**：$$e$$ 处于正规形式当且仅当 $$\neg \exists e'. e \to_\beta e'$$。

某些 lambda 表达式**没有正规形式**——它们会无限归约下去。

### 3.1.7 非终止计算：Ω 组合子 (Non-termination: the Omega Combinator)

最著名的非终止表达式是 $$\Omega$$ 组合子：

$$\Omega = (\lambda x. x x)(\lambda x. x x)$$

让我们跟踪其归约：

```
步骤 1: (λx. x x) (λx. x x)

这里 e₁ = λx. x x, e₂ = λx. x x.
将 e₂ 代入 e₁ 体中的 x:
→β (x x)[x ↦ (λx. x x)]

步骤 2: = (λx. x x) (λx. x x)

我们回到了原表达式！
```

继续归约：

```
步骤 3: →β (x x)[x ↦ (λx. x x)]
       = (λx. x x) (λx. x x)

步骤 4: →β (λx. x x) (λx. x x)

...
```

$$\Omega$$ 归约到自身，无限循环。这是一个完美的最小**发散 (divergent)** 计算——它是 lambda 演算中的等价于 `while(true){}`。$$\Omega$$ 的存在证明了 lambda 演算具有**图灵完备性**所必要的非终止能力。

另一个变体是 $$\Omega_2 = (\lambda x. x x x)(\lambda x. x x x)$$，它的归约序列会增长：

```
步骤 1: (λx. x x x) (λx. x x x)
→β (x x x)[x ↦ (λx. x x x)]
= (λx. x x x) (λx. x x x) (λx. x x x)

步骤 2: →β ((λx. x x x) (λx. x x x)) (λx. x x x)
= (λx. x x x) (λx. x x x) (λx. x x x) (λx. x x x)

表达式每次归约都在增长——这也是发散的，且越变越大。
```

### 3.1.8 正规序求值 (Normal-Order Evaluation)

正则序求值是一种特定的归约策略：**始终选择最外层（最左侧）的 redex**。它的重要性在于：**如果某个表达式有正规形式，正规序求值一定会找到它**。

正规序求值的过程分为四个步骤：

**步骤 1：如果函数体中有 redex，先归约它。**
**步骤 2：执行 α-重命名以避免变量冲突。**
**步骤 3：执行 β-归约。**
**步骤 4：对最终结果递归应用步骤 1-3。**

与**应用序求值 (applicative-order evaluation)**相对比，应用序先归约参数再归约函数体。二者的区别类似于传值调用 (call-by-value) 与传名调用 (call-by-name) 的区别。

**示例：正规序 vs 应用序**

考虑表达式 $$(\lambda x. x) ((\lambda x. x x)(\lambda x. x x))$$：

**正规序求值**：
```
步骤 1: 最外层 redex 是 (λx. x) Ω
        →β x[x ↦ Ω] = Ω
步骤 2: 现在需要归约 Ω
        Ω →β Ω →β Ω → ...  (发散)

但注意: 正规序先选择最外层 redex，已经得到结果 Ω。
```

**应用序求值**：
```
步骤 1: 先归约参数 Ω
        Ω →β Ω →β Ω → ...  (发散了！永远不会到达函数体)
```

正规序在这个例子中至少先产生了一个结果（虽然也是发散的），但应用序在参数求值阶段就永久卡住了。

### 3.1.9 求值策略深度对比 (Evaluation Strategies in Depth)

为了理解不同求值策略的根本差异，我们系统地比较三种策略：

**正规序 (Normal Order)**：始终选择最左最外 redex。归约**从不**发生在抽象体内或参数内部，直到参数被需要。

**应用序 (Applicative Order)**：始终选择最左最内 redex。参数先被完全归约，然后再归约函数调用。

**传值调用 (Call-by-Value)**：与应用序类似，但不在抽象体内归约。大多数编程语言使用此策略。

**示例 1：归约步数比较**

考虑表达式 $$(\lambda x. x x) ((\lambda y. y) (\lambda z. z))$$（此表达式有正规形式）。

**正规序**：
```
1. (λx. x x) ((λy. y)(λz. z))
   →β ((λy. y)(λz. z)) ((λy. y)(λz. z))       [x ↦ ((λy. y)(λz. z))]
2. 需要归约最左最外 redex:
   ((λy. y)(λz. z)) ((λy. y)(λz. z))
   →β (λz. z) ((λy. y)(λz. z))                  [归约左半部的 (λy. y)(λz. z)]
3. (λz. z) ((λy. y)(λz. z))
   →β (λy. y)(λz. z)                             [z ↦ ((λy. y)(λz. z))]
4. (λy. y)(λz. z)
   →β λz. z                                       [y ↦ (λz. z)]
```

正规序用了 4 步归约到正规形式 $$\lambda z. z$$。

**应用序**：
```
1. (λx. x x) ((λy. y)(λz. z))
   先归约参数: (λy. y)(λz. z) →β λz. z
2. (λx. x x) (λz. z)
   →β (λz. z) (λz. z)
3. (λz. z) (λz. z)
   →β λz. z
```

应用序用了 3 步。这似乎说明应用序更高效——但代价是当参数发散时它会卡住（见下一个例子）。

**示例 2：参数不终止的情况**

考虑 $$(\lambda x. \lambda y. y) \Omega$$，其中 $$\Omega = (\lambda x. x x)(\lambda x. x x)$$：

**正规序**：
```
1. (λx. λy. y) Ω
   →β (λy. y)[x ↦ Ω]
   = λy. y
   仅需 1 步即达正规形式！
```

**应用序**：
```
1. 先归约参数 Ω:
   Ω →β Ω →β Ω → ... (无限循环，永远不会归约到函数调用)
```

正规序的优越性在此充分体现：它只在需要参数时才归约它。因为 $$\lambda y. y$$ 根本不使用参数 $$x$$，参数 $$\Omega$$ 从未被归约。

**示例 3：参数被多次使用**

考虑 $$(\lambda x. x x) ((\lambda y. y) 5)$$：

**正规序**：
```
1. (λx. x x) ((λy. y) 5)
   →β ((λy. y) 5) ((λy. y) 5)                  [复制了参数！]
2. 需要归约两个副本:
   左: (λy. y) 5 →β 5
   右: (λy. y) 5 →β 5
3. = 5 5 (这不是合法表达式——正规序失败了，因为它不是纯 lambda 演算的值)
```

**应用序**：
```
1. 先归约参数: (λy. y) 5 →β 5
2. (λx. x x) 5 →β 5 5
   (同上，不是合法值)
```

当参数被多次使用时，应用序只需归约一次参数，而正规序需对每个出现都归约。这是**惰性求值 (lazy evaluation)** 的理论基础——通过共享（thunk/closure memoization）避免重复计算。

`如果实现了共享（如在 Haskell 中），惰性求值结合了两者的优点：它只在需要时才归约参数，且每个参数至多归约一次。但这是通过图归约而不是纯项归约实现的——已超出纯 lambda 演算的范畴。`

### 3.1.10 η-变换 (η-conversion)

除了 α-变换和 β-归约，lambda 演算还有第三条规则：η-变换，它表达了**外延性 (extensionality)**。

**η-归约**：

$$(\lambda x. e x) \to_\eta e \quad (\text{当 } x \notin \mathbf{FV}(e))$$

直觉：如果函数 $$f$$ 对于每个输入 $$x$$ 都等价于 $$e x$$，那么 $$f$$ 等价于 $$e$$。

**η-展开**：

$$e \to_\eta \lambda x. e x \quad (\text{当 } x \notin \mathbf{FV}(e))$$

**示例**：

```
λx. (λy. y) x
→η λy. y
```

因为 $$(\lambda y. y) x \to_\beta x$$，且 $$\lambda x. x$$ 等价于 $$\lambda y. y$$。

η-变换与 β-归约、α-变换一起构成了 lambda 演算的完整等式理论。在类型系统中研究**η-展开**对于理解多态函数的性质至关重要。

### 3.1.11 德布鲁因索引 (De Bruijn Indices)

α-等价性要求我们使用名称来比较 lambda 项，这在实现中很麻烦。**德布鲁因索引 (De Bruijn indices)** 提供了一种语法表示法，通过**自然数**代替变量名，使得 α-等价的项在语法上完全相同。

核心思想：变量由其绑定的抽象和出现位置间的**绑定层数**来表示。数 $$0$$ 表示最近绑定的变量，$$1$$ 表示第二近的，依此类推。自由变量使用超过所有绑定级别的数字。

**从命名到德布鲁因索引的转换示例**：

| 命名表示 (Named) | 德布鲁因表示 (De Bruijn) | 说明 |
|---|---|---|
| $$\lambda x. x$$ | $$\lambda. 0$$ | $$x$$ 指向最近的绑定（只有一层） |
| $$\lambda x. \lambda y. x$$ | $$\lambda. \lambda. 1$$ | $$x$$ 跳过 $$y$$ 的绑定，索引为 1 |
| $$\lambda x. \lambda y. y$$ | $$\lambda. \lambda. 0$$ | $$y$$ 是最近的绑定（索引 0） |
| $$\lambda x. \lambda y. x y$$ | $$\lambda. \lambda. 1\ 0$$ | $$x$$=1, $$y$$=0 |
| $$\lambda f. \lambda x. f (f x)$$ | $$\lambda. \lambda. 1\ (1\ 0)$$ | $$f$$=1, $$x$$=0 |

**Church 数 two 的德布鲁因表示**：

$$\mathbf{two} = \lambda f. \lambda x. f (f x)$$ 变为：
```
λ. λ. 1 (1 0)
```

**β-归约在德布鲁因索引中的实现**更加统一，因为不需要处理变量名冲突（不需要 α-变换！）。当我们需要替换时，会调整自由变量的索引（这称为**移位操作 (lifting)**）。

例如，$$(\lambda. 0\ 1) (\lambda. 0)$$ 的归约（在命名系统中为 $$(\lambda x. x y)(\lambda z. z)$$）：
- $$y$$ 是一个自由变量，索引为 1（跳过最内层绑定）
- 进行 β-归约时，$$0$$ 被替换为参数，自由变量 $$1$$ 保持不变

德布鲁因索引是**显式替换 (explicit substitution)** 理论的基础，也在所有主流 lambda 演算实现中被使用（如 GHC、Coq、Agda 等）。它消除了 α-变换的全部复杂性。

**变量的"自由"判断**在德布鲁因表示中变为：一个数字 $$k$$ 引用的绑定距离如果大于当前位置的绑定器数目，则 $$k$$ 是自由变量。

使用德布鲁因索引，我们可以将替换干净地定义为不含变量捕获问题的操作。但这超出了本章的范围。重点理解：德布鲁因索引**不是**一个不同的计算模型——它是完全相同的 lambda 演算，只是使用了在实现中更方便的表示。在类型理论和证明助手中，德布鲁因表示几乎是唯一的标准实现方式。

## 3.2 Church 编码：用 Lambda 演算编码数据 (Encoding Data)

令人惊叹的是，无需任何内置类型，纯 lambda 演算就能编码布尔值、数对和自然数。这一方案由 Alonzo Church 发明，称为 **Church 编码 (Church encoding)**。

### 3.2.1 布尔值 (Booleans)

Church 编码的布尔值定义为：

$$$$\mathbf{true} = \lambda t. \lambda f. t$$$$

$$$$\mathbf{false} = \lambda t. \lambda f. f$$$$

直观解释：$$\mathbf{true}$$ 是两个参数中选择第一个的函数，$$\mathbf{false}$$ 是选择第二个的函数。任何 Church 布尔值都是一个二元选择函数。

在此基础上定义逻辑运算：

**与 (and)**：

$$$$\mathbf{and} = \lambda a. \lambda b. a b \mathbf{false}$$$$

**或 (or)**：

$$$$\mathbf{or} = \lambda a. \lambda b. a \mathbf{true} b$$$$

**非 (not)**：

$$$$\mathbf{not} = \lambda a. a \mathbf{false} \mathbf{true}$$$$

**条件 (if)**：

$$$$\mathbf{if} = \lambda c. \lambda t. \lambda e. c t e$$$$

---

**推导：$$\mathbf{and\ true\ false} \to_\beta \mathbf{false}$$**

```
and true false
= (λa. λb. a b false) true false
→β (λb. a b false)[a ↦ true] false
= (λb. true b false) false
→β (true b false)[b ↦ false]
= true false false
= (λt. λf. t) false false
→β (λf. t)[t ↦ false] false
= (λf. false) false
→β false[f ↦ false]
= false
     ✓
```

**推导：$$\mathbf{and\ false\ true} \to_\beta \mathbf{false}$$**

```
and false true
= (λa. λb. a b false) false true
→β (λb. a b false)[a ↦ false] true
= (λb. false b false) true
→β (false b false)[b ↦ true]
= false true false
= (λt. λf. f) true false
→β (λf. f)[t ↦ true] false
= (λf. f) false
→β f[f ↦ false]
= false
     ✓
```

**推导：$$\mathbf{and\ true\ true} \to_\beta \mathbf{true}$$**

```
and true true
= (λa. λb. a b false) true true
→β (λb. true b false) true
→β true true false
= (λt. λf. t) true false
→β (λf. t)[t ↦ true] false
= (λf. true) false
→β true[f ↦ false]
= true
     ✓
```

**完整真值表验证**：

| $$a$$ | $$b$$ | $$\mathbf{and}\ a\ b$$ |
|---------|---------|---------------------------|
| false | false | $$\mathbf{false}$$ |
| false | true | $$\mathbf{false}$$ |
| true | false | $$\mathbf{false}$$ |
| true | true | $$\mathbf{true}$$ |

---

**推导：$$\mathbf{or\ false\ true} \to_\beta \mathbf{true}$$**

```
or false true
= (λa. λb. a true b) false true
→β (λb. a true b)[a ↦ false] true
= (λb. false true b) true
→β (false true b)[b ↦ true]
= false true true
= (λt. λf. f) true true
→β (λf. f)[t ↦ true] true
= (λf. f) true
→β f[f ↦ true]
= true
     ✓
```

**推导：$$\mathbf{or\ false\ false} \to_\beta \mathbf{false}$$**

```
or false false
= (λa. λb. a true b) false false
→β (λb. false true b) false
→β false true false
= (λt. λf. f) true false
→β (λf. f)[t ↦ true] false
= (λf. f) false
→β f[f ↦ false]
= false
     ✓
```

---

**推导：$$\mathbf{not\ true} \to_\beta \mathbf{false}$$**

```
not true
= (λa. a false true) true
→β (a false true)[a ↦ true]
= true false true
= (λt. λf. t) false true
→β (λf. t)[t ↦ false] true
= (λf. false) true
→β false[f ↦ true]
= false
     ✓
```

**推导：$$\mathbf{not\ false} \to_\beta \mathbf{true}$$**

```
not false
= (λa. a false true) false
→β false false true
= (λt. λf. f) false true
→β (λf. f)[t ↦ false] true
= (λf. f) true
→β f[f ↦ true]
= true
     ✓
```

---

**推导：$$\mathbf{if\ true\ A\ B} \to_\beta A$$**

```
if true A B
= (λc. λt. λe. c t e) true A B
→β (λt. λe. c t e)[c ↦ true] A B
= (λt. λe. true t e) A B
→β (λe. true t e)[t ↦ A] B
= (λe. true A e) B
→β (true A e)[e ↦ B]
= true A B
= (λt. λf. t) A B
→β (λf. t)[t ↦ A] B
= (λf. A) B
→β A[f ↦ B]
= A
     ✓
```

### 3.2.2 数对 (Pairs)

Church 编码的数对定义为：

$$$$\mathbf{pair} = \lambda x. \lambda y. \lambda s. s x y$$$$

$$$$\mathbf{first} = \lambda p. p \mathbf{true}$$$$

$$$$\mathbf{second} = \lambda p. p \mathbf{false}$$$$

**推导：$$\mathbf{first\ (pair\ A\ B)} \to_\beta A$$**

```
first (pair A B)
= (λp. p true) (pair A B)
→β (p true)[p ↦ (pair A B)]
= (pair A B) true
= ((λx. λy. λs. s x y) A B) true

先归约 pair A B:
(λx. λy. λs. s x y) A B
→β (λy. λs. s x y)[x ↦ A] B
= (λy. λs. s A y) B
→β (λs. s A y)[y ↦ B]
= λs. s A B

继续:
(λs. s A B) true
→β (s A B)[s ↦ true]
= true A B
→β A  [如上节所示]
     ✓
```

**推导：$$\mathbf{second\ (pair\ A\ B)} \to_\beta B$$**

```
second (pair A B)
= (λp. p false) (pair A B)
→β (pair A B) false
= (λs. s A B) false
→β (s A B)[s ↦ false]
= false A B
= (λt. λf. f) A B
→β (λf. f)[t ↦ A] B
= (λf. f) B
→β f[f ↦ B]
= B
     ✓
```

### 3.2.3 Church 数 (Church Numerals)

Church 对自然数的编码基于一个核心直觉：**自然数 $$n$$ 就是一个对函数 $$f$$ 应用 $$n$$ 次的行为**。

$$$$\mathbf{zero} = \lambda f. \lambda x. x$$$$

$$$$\mathbf{one} = \lambda f. \lambda x. f x$$$$

$$$$\mathbf{two} = \lambda f. \lambda x. f (f x)$$$$

$$$$\mathbf{three} = \lambda f. \lambda x. f (f (f x))$$$$

通用模式：

$$\mathbf{n} = \lambda f. \lambda x. \underbrace{f (f (\dots (f}_{n \text{次}} x)\dots))$$

---

**后继函数 $$\mathbf{incr}$$**：

$$$$\mathbf{incr} = \lambda n. \lambda f. \lambda x. f (n f x)$$$$

理解：$$\mathbf{incr}$$ 接受一个 Church 数 $$n$$，返回一个行为为"先应用 $$n$$ 次 $$f$$ 再应用一次 $$f$$"的函数。

**推导：$$\mathbf{incr\ zero} \to_\beta \mathbf{one}$$**

```
incr zero
= (λn. λf. λx. f (n f x)) zero
→β (λf. λx. f (n f x))[n ↦ zero]
= λf. λx. f (zero f x)

展开 zero f x:
zero f x
= (λf. λx. x) f x
→β (λx. x)[f ↦ f] x
= (λx. x) x
→β x[x ↦ x]
= x

代回:
= λf. λx. f x
= one
     ✓
```

**推导：$$\mathbf{incr\ one} \to_\beta \mathbf{two}$$**

```
incr one
= (λn. λf. λx. f (n f x)) one
→β λf. λx. f (one f x)

展开 one f x:
one f x
= (λf. λx. f x) f x
→β (λx. f x)[f ↦ f] x
= (λx. f x) x
→β (f x)[x ↦ x]
= f x

代回:
= λf. λx. f (f x)
= two
     ✓
```

---

**加法 $$\mathbf{plus}$$**：

$$$$\mathbf{plus} = \lambda m. \lambda n. m \mathbf{incr} n$$$$

加法的直觉：将 $$\mathbf{incr}$$ 应用到第二个数上总共 $$m$$ 次。

**推导：$$\mathbf{plus\ two\ three} \to_\beta \mathbf{five}$$**

这是本章最长的单条推导。让我们一步步来：

```
plus two three
= (λm. λn. m incr n) two three
```

```
步骤 1: 将 two 代入 m
→β (λn. m incr n)[m ↦ two] three
= (λn. two incr n) three
```

```
步骤 2: 将 three 代入 n
→β (two incr n)[n ↦ three]
= two incr three
```

```
步骤 3: 展开 two
= (λf. λx. f (f x)) incr three
```

```
步骤 4: 将 incr 代入 f
→β (λx. f (f x))[f ↦ incr] three
= (λx. incr (incr x)) three
```

```
步骤 5: 将 three 代入 x
→β (incr (incr x))[x ↦ three]
= incr (incr three)
```

```
步骤 6: 先计算内层 incr three
incr three
= (λn. λf. λx. f (n f x)) three
→β λf. λx. f (three f x)
= λf. λx. f (f (f (f x)))    [three f x = f(f(f x)), 再加一层 f]
= four
```

```
步骤 7: 计算 incr four
incr four
= (λn. λf. λx. f (n f x)) four
→β λf. λx. f (four f x)
= λf. λx. f (f (f (f (f x))))
= five
```

因此：

$$\mathbf{plus\ two\ three} \to_\beta^* \mathbf{five}$$

---

**推导：$$\mathbf{plus\ one\ two} \to_\beta \mathbf{three}$$**

```
plus one two
= (λm. λn. m incr n) one two
→β (λn. one incr n) two
→β one incr two
= (λf. λx. f x) incr two
→β (λx. incr x) two
→β incr two
→β λf. λx. f (two f x)
= λf. λx. f (f (f x))
= three
     ✓
```

---

**乘法 $$\mathbf{times}$$**：

$$$$\mathbf{times} = \lambda m. \lambda n. m (\mathbf{plus}\ n) \mathbf{zero}$$$$

直觉：$$m \times n$$ 是将 $$n$$ 累加 (plus n) 这个操作执行 $$m$$ 次，起始值为 $$\mathbf{zero}$$。

另一种等价定义：

$$$$\mathbf{times} = \lambda m. \lambda n. \lambda f. m (n f)$$$$

**推导：$$\mathbf{times\ two\ three} \to_\beta \mathbf{six}$$**（使用第二种定义）

```
times two three
= (λm. λn. λf. m (n f)) two three
→β (λn. λf. m (n f))[m ↦ two] three
= (λn. λf. two (n f)) three
→β (λf. two (n f))[n ↦ three]
= λf. two (three f)
```

```
展开 two (three f):
two (three f)
= (λf. λx. f (f x)) (three f)
→β (λx. f (f x))[f ↦ (three f)]
= λx. (three f) ((three f) x)

现在 (three f) x = f (f (f x))
所以 (three f) ((three f) x) = f (f (f (f (f (f x)))))

因此:
= λx. f (f (f (f (f (f x)))))
```

```
最终的表达式:
λf. λx. f (f (f (f (f (f x)))))
= six
     ✓
```

---

**零测试 $$\mathbf{iszero}$$**：

$$$$\mathbf{iszero} = \lambda n. n (\lambda x. \mathbf{false}) \mathbf{true}$$$$

直觉：将函数 $$(\lambda x. \mathbf{false})$$ 应用到 $$\mathbf{true}$$ 上共 $$n$$ 次。如果 $$n = 0$$，零次应用得到 $$\mathbf{true}$$。如果 $$n > 0$$，至少应用一次后得到 $$\mathbf{false}$$。

**推导：$$\mathbf{iszero\ zero} \to_\beta \mathbf{true}$$**

```
iszero zero
= (λn. n (λx. false) true) zero
→β (n (λx. false) true)[n ↦ zero]
= zero (λx. false) true
= (λf. λx. x) (λx. false) true
→β (λx. x)[f ↦ (λx. false)] true
= (λx. x) true
→β x[x ↦ true]
= true
     ✓
```

**推导：$$\mathbf{iszero\ one} \to_\beta \mathbf{false}$$**

```
iszero one
= (λn. n (λx. false) true) one
→β one (λx. false) true
= (λf. λx. f x) (λx. false) true

步骤 1: (λf. λx. f x) (λx. false)
→β (λx. f x)[f ↦ (λx. false)]
= λx. (λx. false) x

步骤 2: (λx. (λx. false) x) true
→β ((λx. false) x)[x ↦ true]
= (λx. false) true

步骤 3: (λx. false) true
→β false[x ↦ true]
= false
     ✓
```

**推导：$$\mathbf{iszero\ two} \to_\beta \mathbf{false}$$**

```
iszero two
= (λn. n (λx. false) true) two
→β two (λx. false) true
= (λf. λx. f (f x)) (λx. false) true

步骤 1: 将 (λx. false) 代入 f
→β (λx. f (f x))[f ↦ (λx. false)] true
= (λx. (λx. false) ((λx. false) x)) true

步骤 2: 将 true 代入 x
→β (λx. false) ((λx. false) true)
→β false[x ↦ ((λx. false) true)]
= false
     ✓
```

#### 前驱函数 $$\mathbf{pred}$$ (Predecessor)

前驱函数是 Church 编码中难度最高的算术运算。核心思路是：从 $$\mathbf{zero}$$ 和 $$(0, 0)$$ 开始，$$n$$ 次应用变换 $$(a, b) \mapsto (b, b+1)$$，其中对第一个分量不增 1 相当于始终比后继"慢一步"。

定义辅助的数对变换：

$$$$\mathbf{shift} = \lambda p. \mathbf{pair}\ (\mathbf{second}\ p)\ (\mathbf{incr}\ (\mathbf{second}\ p))$$$$

然后：

$$$$\mathbf{pred} = \lambda n. \mathbf{first}\ (n\ \mathbf{shift}\ (\mathbf{pair}\ \mathbf{zero}\ \mathbf{zero}))$$$$

**推导：$$\mathbf{pred\ zero} \to_\beta \mathbf{zero}$$**

```
pred zero
= (λn. first (n shift (pair zero zero))) zero
→β first (zero shift (pair zero zero))

zero shift (pair zero zero)
= (λf. λx. x) shift (pair zero zero)
→β (λx. x) (pair zero zero)
→β pair zero zero

first (pair zero zero)
→β zero
     ✓
```

**推导：$$\mathbf{pred\ one} \to_\beta \mathbf{zero}$$**

```
pred one
= (λn. first (n shift (pair zero zero))) one
→β first (one shift (pair zero zero))

one shift (pair zero zero)
= (λf. λx. f x) shift (pair zero zero)
→β (λx. shift x) (pair zero zero)
→β shift (pair zero zero)

shift (pair 0 0)
= (λp. pair (second p) (incr (second p))) (pair 0 0)
→β pair (second (pair 0 0)) (incr (second (pair 0 0)))
→β pair 0 (incr 0)
→β pair 0 1

first (pair 0 1)
→β 0 = zero
     ✓
```

**推导：$$\mathbf{pred\ two} \to_\beta \mathbf{one}$$**

```
pred two
= first (two shift (pair zero zero))

two shift (pair zero zero)
= (λf. λx. f (f x)) shift (pair 0 0)
→β (λx. shift (shift x)) (pair 0 0)
→β shift (shift (pair 0 0))

先算内层 shift (pair 0 0):
shift (pair 0 0) →β pair 0 1

再算外层 shift (pair 0 1):
shift (pair 0 1)
→β pair (second (pair 0 1)) (incr (second (pair 0 1)))
→β pair 1 (incr 1)
→β pair 1 2

first (pair 1 2) →β 1 = one
     ✓
```

#### Church 减法 (Subtraction)

有了前驱函数，减法是对 $$m$$ 次前驱操作应用于 $$n$$：

$$$$\mathbf{sub} = \lambda m. \lambda n. n\ \mathbf{pred}\ m$$$$

**推导：$$\mathbf{sub\ three\ two} \to_\beta \mathbf{one}$$**

```
sub three two
= (λm. λn. n pred m) three two
→β (λn. n pred three) two
→β two pred three
= (λf. λx. f (f x)) pred three
→β (λx. pred (pred x)) three
→β pred (pred three)
→β pred two
→β one
     ✓
```

#### Church 相等测试 (Equality Test)

利用减法和零测试，可以定义相等测试：

$$$$\mathbf{eq} = \lambda m. \lambda n. \mathbf{and}\ (\mathbf{iszero}\ (\mathbf{sub}\ m\ n))\ (\mathbf{iszero}\ (\mathbf{sub}\ n\ m))$$$$

由于 Church 数不能表示负数，$$\mathbf{sub}\ a\ b$$ 当 $$a < b$$ 时返回 $$\mathbf{zero}$$。因此 $$m = n$$ 当且仅当 $$m - n = 0$$ 且 $$n - m = 0$$。

**推导：$$\mathbf{eq\ two\ two} \to_\beta \mathbf{true}$$**

```
eq two two
= and (iszero (sub two two)) (iszero (sub two two))
= and (iszero zero) (iszero zero)
= and true true
→β true
     ✓
```

**推导：$$\mathbf{eq\ one\ two} \to_\beta \mathbf{false}$$**

```
eq one two
= and (iszero (sub one two)) (iszero (sub two one))
= and (iszero zero) (iszero one)
= and true false
→β false
     ✓
```

#### Church 链表 (Church Lists)

Church 编码也可以表示链表（类似于 Lisp 的 `cons` 列表）：

$$$$\mathbf{nil} = \lambda c. \lambda n. n$$$$

$$$$\mathbf{cons} = \lambda h. \lambda t. \lambda c. \lambda n. c h (t c n)$$$$

直觉：链表表示为一个折叠操作。$$\mathbf{nil}$$ 忽略构造器 $$c$$ 并返回底值 $$n$$；$$\mathbf{cons}$$ 对头元素应用 $$c$$，然后对尾链表递归折叠。

**列表折叠函数**：

$$$$\mathbf{fold} = \lambda l. \lambda c. \lambda n. l c n$$$$

**示例**：编码链表 $$\mathbf{[true, false]}$$：

```
[true, false]
= cons true (cons false nil)
= (λh. λt. λc. λn. c h (t c n)) true ((λh. λt. λc. λn. c h (t c n)) false nil)
```

验证（部分归约略去细节）：
```
fold [true, false] (λx. x) nil
→ ... → true
```

Church 链表是 Scott 编码的特例，展示了纯 lambda 演算如何在无内置数据结构的情况下模拟任意代数数据类型。

#### 组合子逻辑 (Combinatory Logic) — 简述

与 lambda 演算紧密相关的另一种形式体系是**组合子逻辑 (combinatory logic)**，由 Moses Schonfinkel 和 Haskell Curry 独立发展。组合子逻辑通过少量固定的**组合子**和**应用**来定义所有可计算函数——完全消除了 lambda 演算中的变量与绑定。

两个最基本组合子是：

$$$$\mathbf{S} = \lambda x. \lambda y. \lambda z. x z (y z)$$$$

$$$$\mathbf{K} = \lambda x. \lambda y. x$$$$

从 $$\mathbf{S}$$ 和 $$\mathbf{K}$$ 出发，所有可计算函数都能被构造——不需要变量或抽象。例如：

- 恒等函数：$$\mathbf{I} = \mathbf{S K K}$$（验证：$$\mathbf{S K K}\ x = \mathbf{K} x (\mathbf{K} x) = x$$）
- 函数组合：$$\mathbf{B} = \mathbf{S (K S) K}$$（$$\mathbf{B}\ f\ g\ x = f (g x)$$）

**SK 组合子演算**的非凡之处在于：它表明**变量根本不是计算的必需品**——计算可以通过无名称的函数组合实现。这一洞见对实现高效的图归约机有直接应用，也是 Miranda 编译器和 Haskell G-machine 等实际系统的理论基础。

组合子逻辑与 lambda 演算通过**括号抽象 (bracket abstraction)** 算法建立对应关系：将任何 lambda 表达式翻译为只含 $$\mathbf{S}$$、$$\mathbf{K}$$ 和 $$\mathbf{I}$$ 的组合子表达式。这一翻译的核心意义在于证明了"变量是可消除的"——尽管在实践中，完全消除变量的代价往往是计算长度的指数爆炸。

### 3.2.4 递归与不动点组合子 (Recursion and the Y Combinator)

在 lambda 演算中实现递归需要克服一个基本困难：**匿名函数没有名字，无法直接引用自身**。

#### 自传递技巧

一个聪明的技巧是让函数接收**自身**作为参数：

```
fact := λf. λn. if (iszero n) one (times n (f f (pred n)))
```

调用时：
```
fact fact 5
```

但我们需要一个**不动点组合子**来更优雅地表达递归。

#### Y 组合子的推导

**步骤 1：从阶乘函数出发**

我们希望定义：
```
fact n = if (iszero n) one (times n (fact (pred n)))
```

**步骤 2：消除自引用——添加额外的函数参数**

将自引用抽象为一个参数 $$f$$：
```
F := λf. λn. if (iszero n) one (times n (f (pred n)))
```

现在 $$F$$ 不是递归的——它需要一个函数 $$f$$ 作为参数。如果我们将 $$F$$ 的某个合适的不动点传给它，就能获得阶乘函数。

**步骤 3：寻找不动点**

如果存在一个 $$Y$$ 使得 $$Y F = F (Y F)$$，那么 $$Y F$$ 就是阶乘函数。因为：
```
Y F n
= F (Y F) n
= if (iszero n) one (times n ((Y F) (pred n)))
```

这正是我们想要的递归行为。

**步骤 4：构造 Y 组合子**

考虑一个函数，它接受自身作为参数：
```
λx. F (x x)
```

将其应用到自身：
```
(λx. F (x x)) (λx. F (x x))
→β F ((λx. F (x x)) (λx. F (x x)))
```

注意：结果等于 $$F$$ 应用到原表达式上。这就是不动点性质！

因此：
```
Y := λF. (λx. F (x x)) (λx. F (x x))
```

**步骤 5：验证不动点性质**

```
Y F
= (λF. (λx. F (x x)) (λx. F (x x))) F
→β (λx. F (x x)) (λx. F (x x))
→β F ((λx. F (x x)) (λx. F (x x)))
←β F (Y F)
```

因此 $$Y F =_β F (Y F)$$，不动点性质成立。$$\blacksquare$$

**Y 组合子**：

$$$$\mathbf{Y} = \lambda f. (\lambda x. f (x x))(\lambda x. f (x x))$$$$

> **注意**：此形式的 Y 组合子仅适用于传名调用 (call-by-name) 或正规序求值。在传值调用 (call-by-value) 中需要使用 Z 组合子：$$Z = \lambda f. (\lambda x. f (\lambda v. x x v)) (\lambda x. f (\lambda v. x x v))$$。

#### 推导 Y 组合子计算 $$\mathbf{Y\ F\ 3}$$

设 $$F$$ 为阶乘函数的函数版本：

$$F = \lambda f. \lambda n. \mathbf{if}\ (\mathbf{iszero}\ n)\ \mathbf{one}\ (\mathbf{times}\ n\ (f\ (\mathbf{pred}\ n)))$$

为了简化，我们假设 $$\mathbf{pred\ three} = \mathbf{two}$$，$$\mathbf{pred\ two} = \mathbf{one}$$，$$\mathbf{pred\ one} = \mathbf{zero}$$，$$\mathbf{times\ m\ n} = \mathbf{m} \times \mathbf{n}$$。

推导 $$\mathbf{Y\ F\ 3}$$：

```
1. Y F 3
   = (λf. (λx. f (x x))(λx. f (x x))) F 3

2. →β (λx. F (x x))(λx. F (x x)) 3
      [f ↦ F]

   令 G = (λx. F (x x))(λx. F (x x))
   则 = G 3

3. 展开 G:
   G = (λx. F (x x))(λx. F (x x))
     →β F ((λx. F (x x))(λx. F (x x)))
     = F G

   因此 G 3 = F G 3

4. F G 3
   = (λf. λn. if (iszero n) one (times n (f (pred n)))) G 3
   →β (λn. if (iszero n) one (times n (G (pred n)))) 3
   →β if (iszero 3) one (times 3 (G (pred 3)))

5. iszero 3 →β false
   = if false one (times 3 (G 2))
   →β times 3 (G 2)

6. 需要计算 G 2:
   G 2 = F G 2
   = (λn. if (iszero n) one (times n (G (pred n)))) 2
   →β if false one (times 2 (G 1))
   →β times 2 (G 1)

7. G 1 = F G 1
   →β if false one (times 1 (G 0))
   →β times 1 (G 0)

8. G 0 = F G 0
   →β if (iszero 0) one (times 0 (G (pred 0)))
   →β if true one (times 0 ...)
   →β one

9. 回代:
   G 1 = times 1 one = one
   G 2 = times 2 one = two
   G 3 = times 3 two = six
```

因此 $$\mathbf{Y\ F\ 3} \to_\beta^* \mathbf{six}$$，即 $$3! = 6$$。$$\blacksquare$$

#### 传值调用下的 Z 组合子 (Z Combinator for Call-by-Value)

在传值调用 (call-by-value) 语言（如 Scheme、ML、JavaScript）中，$$x x$$ 形式的表达式会被无限展开，因为参数在传入前必须先求值。简单的 Y 组合子会导致无穷回归。

**Z 组合子**将自应用包装在抽象中，延迟求值：

$$$$\mathbf{Z} = \lambda f. (\lambda x. f (\lambda v. x x v)) (\lambda x. f (\lambda v. x x v))$$$$

验证不动点性质（注意额外的 $$\lambda v$$ 层）：

```
Z F
= (λf. (λx. f (λv. x x v))(λx. f (λv. x x v))) F
→β (λx. F (λv. x x v))(λx. F (λv. x x v))
→β F (λv. (λx. F (λv. x x v))(λx. F (λv. x x v)) v)
= F (λv. (Z F) v)
```

在传值调用下，$$\lambda v. (Z F) v$$ 被视为值（等待应用的函数），不会进一步展开。当真正的参数到来时：

```
F (λv. (Z F) v) 3
= F ((Z F) 3)

注意: λv. (Z F) v 应用到 3 时:
(λv. (Z F) v) 3 →β (Z F) 3 = F ((Z F) 3)
然后 F 应用到结果继续递归。
```

Z 组合子在所有严格求值语言中工作。Haskell 因为使用惰性求值，所以 Y 组合子即可使用。

#### 递归函数的更多示例

**斐波那契 (Fibonacci)**：

定义 Fibonacci 生成函数：

$$\mathbf{fibgen} = \lambda f. \lambda n. \mathbf{if}\ (\mathbf{or}\ (\mathbf{iszero}\ n)\ (\mathbf{iszero}\ (\mathbf{pred}\ n)))\ \mathbf{one}\ (\mathbf{plus}\ (f\ (\mathbf{pred}\ n))\ (f\ (\mathbf{pred}\ (\mathbf{pred}\ n))))$$

则：
$$\mathbf{fib} = \mathbf{Y}\ \mathbf{fibgen}$$

**推导 $$\mathbf{fib\ two} \to_\beta^* \mathbf{one}$$** （简化版）：

```
fib two
= Y fibgen two
= fibgen (Y fibgen) two
= fibgen fib two
= if (or (iszero 2) (iszero (pred 2))) one (plus (fib (pred 2)) (fib (pred (pred 2))))
= if (or false (iszero 1)) one (plus (fib 1) (fib 0))
= ... 
→β plus (fib 1) (fib 0)
→β plus one one
→β two
```

(Fibonacci 的 Church 数: fib(0)=1, fib(1)=1, fib(2)=2。所以 fib two = two。注意这里定义为 fib(0)=one 的传统。)

## 3.3 等价计算模型 (Equivalent Models of Computation)

### 3.3.1 图灵机 (Turing Machine)

图灵机是与 lambda 演算等价的计算模型。图灵机由以下组件构成：

1. **纸带 (Tape)**：一条两端无限延伸的线性存储带，划分为离散的格子，每个格子存储一个符号（来自有限字母表）。

2. **磁头 (Head)**：读取/写入当前位置的符号，并可向左或向右移动一格。

3. **状态寄存器 (State Register)**：存储图灵机的当前状态（来自有限状态集）。

4. **指令表 (Instruction Table)**：一组形如 $$(当前状态, 读取符号) \rightarrow (写入符号, 移动方向, 新状态)$$ 的规则。

图灵机是确定性的：对于同一个（状态，符号）对，至多有一条指令。如果不存在匹配的指令，机器**停机 (halt)**。

**形式定义**：图灵机是一个七元组 $$M = (Q, \Sigma, \Gamma, \delta, q_0, q_{\text{accept}}, q_{\text{reject}})$$，其中：
- $$Q$$ 是有限状态集
- $$\Sigma$$ 是输入字母表（不包含空格符号）
- $$\Gamma$$ 是纸带字母表（$$\Sigma \subseteq \Gamma$$，包含空格符号）
- $$\delta: Q \times \Gamma \rightarrow Q \times \Gamma \times \{L, R\}$$ 是转移函数
- $$q_0 \in Q$$ 是初始状态
- $$q_{\text{accept}} \in Q$$ 是接受状态
- $$q_{\text{reject}} \in Q$$ 是拒绝状态（$$q_{\text{accept}} \neq q_{\text{reject}}$$）

### 3.3.2 Church-Turing 论题 (Church-Turing Thesis)

**Church-Turing 论题**（非数学定理，而是关于物理世界可计算性的假设）：

> 任何在物理世界中可有效计算的函数，都可以被图灵机（或等价的 lambda 演算）计算。

论题的等价表述包括：
- 所有合理的计算模型都是等价的（互模拟的）
- lambda 演算、图灵机、递归函数、Post 系统、马尔可夫算法等都能计算完全相同的函数类
- 不存在能够计算图灵不可计算函数的物理计算装置（在经典物理框架内）

论题的核心洞见：**计算的本质不依赖于特定的实现机制**。无论是机械的图灵机、符号的 lambda 演算，还是现代的电子计算机，它们计算能力的边界是相同的。

### 3.3.3 图灵完备性 (Turing Completeness)

一个计算模型（或编程语言）是**图灵完备的**当且仅当它能模拟任何图灵机（或能计算任何一般递归函数）。

图灵完备性的**充要条件**（通常在编程语言中验证的方法）：

1. 能够表示**无限内存**（lambda 演算中通过函数调用的无界嵌套实现；语言中通过可增长的堆/栈实现）
2. 具有**条件分支**（lambda 演算中通过 Church 布尔值实现）
3. 具有**重复/迭代/递归**的能力（lambda 演算中通过不动点组合子实现）

**重要事实**：几乎所有通用编程语言都是图灵完备的。即便是表面简单的系统也可能图灵完备——例如 Conway 的生命游戏、Rule 110（元胞自动机）、甚至某些宏展开系统（如 C++ 模板）。

### 3.3.4 Lambda 演算与图灵机的等价性 (Equivalence Proof Sketch)

虽然完整的等价性证明超出了本章范围，但其结构值得概述：

**方向 1：图灵机模拟 Lambda 演算**。为每个图灵机状态构造一个 Church 编码。纸带可以表示为两个列表（左半部分和右半部分），转移函数表示为 Church 布尔条件链。这证明 lambda 演算至少如图灵机强大。

**方向 2：Lambda 演算模拟图灵机**。定义图灵机对 lambda 项的归约操作。由于 lambda 演算具有并发性质（多个 redex 可以同时存在），而图灵机是顺序的，模拟需要实现一个归约策略——正规序恰好是最安全的选择，因为它保证存在正规形式时能找到。

**结论**：lambda 演算和图灵机计算完全相同的函数类——即**一般递归函数 (general recursive functions)** 类。这个类恰好是"可计算函数"的形式化。

### 3.3.5 停机问题 (The Halting Problem)

一个自然的问题是：**是否所有 lambda 表达式都有正规形式？** 答案是**否定的**——而且更根本的是，**不存在一个通用的算法**能判定任意 lambda 表达式是否会终止。

这就是**停机问题 (Halting Problem)**：不存在一个 Church 编码的函数 $$H$$ 使得对任意 lambda 项 $$M$$：

$$H\ M = \begin{cases} \mathbf{true} & \text{如果 } M \text{ 有正规形式} \\ \mathbf{false} & \text{如果 } M \text{ 没有正规形式} \end{cases}$$

证明采用标准的对角线论证法：假设 $$H$$ 存在，构造一个矛盾的表达式 $$D$$：

$$D = \lambda x. \mathbf{if}\ (H\ x)\ \Omega\ \mathbf{true}$$

其中 $$\Omega$$ 是不终止组合子。现在考虑 $$H\ D$$：

- 如果 $$H\ D = \mathbf{true}$$，则 $$D\ D = \mathbf{if}\ \mathbf{true}\ \Omega\ \mathbf{true} \to_\beta \Omega$$（不终止），矛盾。
- 如果 $$H\ D = \mathbf{false}$$，则 $$D\ D = \mathbf{if}\ \mathbf{false}\ \Omega\ \mathbf{true} \to_\beta \mathbf{true}$$（终止），矛盾。

因此 $$H$$ 不存在。停机问题的不可判定性是计算理论最根本的结果之一。$$\blacksquare$$

## 3.4 操作语义 (Operational Semantics)

操作语义描述了程序的**执行过程**——即程序如何在抽象机器上一步步归约。我们将定义一种简单命令式语言的大步操作语义（自然语义）。

### 3.4.1 简单命令式语言 (Simple Imperative Language)

语法：

```
e ::= x          变量
    | n          整数常量
    | e₁ + e₂    加法
    | e₁ > e₂    大于比较
    | e₁ & e₂    逻辑与 (短路求值)
    | b          布尔常量

b ::= true | false

s ::= skip            空语句
    | x = e           赋值
    | s₁ ; s₂         顺序
    | if e s₁ s₂      条件
    | while e s       循环
```

### 3.4.2 状态与转移 (States and Transitions)

**状态 (State)**：$$\sigma: \mathbf{Var} \rightarrow \mathbf{Value}$$，是从变量名到值的映射。我们用 $$\sigma(x)$$ 查找变量 $$x$$ 在新状态 $$\{x \mapsto v, y \mapsto w, \dots\}$$ 中的值。

**转移关系**分为两级：

- **表达式求值**：$$\langle e, \sigma \rangle \Downarrow \langle v, \sigma' \rangle$$

  在状态 $$\sigma$$ 下，表达式 $$e$$ 求值为值 $$v$$（求值可能产生副作用从而产生新状态 $$\sigma'$$，但在纯表达式中 $$\sigma' = \sigma$$）。

- **语句执行**：$$\langle s, \sigma \rangle \Downarrow \langle u, \sigma' \rangle$$

  在状态 $$\sigma$$ 下，语句 $$s$$ 执行后得到新状态 $$\sigma'$$。$$u$$ 是 unit 值（记作 $$\mathbf{skip}$$）。

### 3.4.3 表达式的大步语义规则

**字面量规则 (Literal)**：

$$
\begin{array}{c}
\hline
\langle n, \sigma \rangle \Downarrow \langle n, \sigma \rangle
\end{array}
$$

整数常量求值为其自身，状态不变。

$$
\begin{array}{c}
\hline
\langle b, \sigma \rangle \Downarrow \langle b, \sigma \rangle
\end{array}
$$

**变量规则 (Variable)**：

$$
\begin{array}{c}
\hline
\langle x, \sigma \rangle \Downarrow \langle \sigma(x), \sigma \rangle
\end{array}
$$

**加法规则 (Addition)**——标准（左到右）求值：

$$
\begin{array}{c}
\langle e_1, \sigma \rangle \Downarrow \langle n_1, \sigma_1 \rangle \quad
\langle e_2, \sigma_1 \rangle \Downarrow \langle n_2, \sigma_2 \rangle \\
\hline
\langle e_1 + e_2, \sigma \rangle \Downarrow \langle n_1 + n_2, \sigma_2 \rangle
\end{array}
$$

**加法规则**——右到左求值：

$$
\begin{array}{c}
\langle e_2, \sigma \rangle \Downarrow \langle n_2, \sigma_1 \rangle \quad
\langle e_1, \sigma_1 \rangle \Downarrow \langle n_1, \sigma_2 \rangle \\
\hline
\langle e_1 + e_2, \sigma \rangle \Downarrow \langle n_1 + n_2, \sigma_2 \rangle
\end{array}
$$

**比较规则 (Greater Than)**：

$$
\begin{array}{c}
\langle e_1, \sigma \rangle \Downarrow \langle n_1, \sigma_1 \rangle \quad
\langle e_2, \sigma_1 \rangle \Downarrow \langle n_2, \sigma_2 \rangle \\
\hline
\langle e_1 > e_2, \sigma \rangle \Downarrow \langle n_1 > n_2, \sigma_2 \rangle
\end{array}
$$

**逻辑与规则 (Short-Circuit Conjunction)**：

$$
\begin{array}{c}
\langle e_1, \sigma \rangle \Downarrow \langle \mathbf{false}, \sigma_1 \rangle \\
\hline
\langle e_1 \,\&\, e_2, \sigma \rangle \Downarrow \langle \mathbf{false}, \sigma_1 \rangle
\end{array}
$$

$$
\begin{array}{c}
\langle e_1, \sigma \rangle \Downarrow \langle \mathbf{true}, \sigma_1 \rangle \quad
\langle e_2, \sigma_1 \rangle \Downarrow \langle v, \sigma_2 \rangle \\
\hline
\langle e_1 \,\&\, e_2, \sigma \rangle \Downarrow \langle v, \sigma_2 \rangle
\end{array}
$$

注意到关键特性：如果 $$e_1$$ 求值为 $$\mathbf{false}$$，$$e_2$$ **不需求值**——这就是短路行为。

### 3.4.4 语句的大步语义规则

**空语句 (Skip)**：

$$
\begin{array}{c}
\hline
\langle \mathbf{skip}, \sigma \rangle \Downarrow \langle \mathbf{skip}, \sigma \rangle
\end{array}
$$

**赋值 (Assignment)**：

$$
\begin{array}{c}
\langle e, \sigma \rangle \Downarrow \langle v, \sigma' \rangle \\
\hline
\langle x = e, \sigma \rangle \Downarrow \langle \mathbf{skip}, \sigma'\{x \mapsto v\} \rangle
\end{array}
$$

**顺序 (Sequence)**：

$$
\begin{array}{c}
\langle s_1, \sigma \rangle \Downarrow \langle \mathbf{skip}, \sigma_1 \rangle \quad
\langle s_2, \sigma_1 \rangle \Downarrow \langle u, \sigma_2 \rangle \\
\hline
\langle s_1 ; s_2, \sigma \rangle \Downarrow \langle u, \sigma_2 \rangle
\end{array}
$$

**条件 (Conditional)**：

$$
\begin{array}{c}
\langle e, \sigma \rangle \Downarrow \langle \mathbf{true}, \sigma_1 \rangle \quad
\langle s_1, \sigma_1 \rangle \Downarrow \langle u, \sigma_2 \rangle \\
\hline
\langle \mathbf{if}\ e\ s_1\ s_2, \sigma \rangle \Downarrow \langle u, \sigma_2 \rangle
\end{array}
$$

$$
\begin{array}{c}
\langle e, \sigma \rangle \Downarrow \langle \mathbf{false}, \sigma_1 \rangle \quad
\langle s_2, \sigma_1 \rangle \Downarrow \langle u, \sigma_2 \rangle \\
\hline
\langle \mathbf{if}\ e\ s_1\ s_2, \sigma \rangle \Downarrow \langle u, \sigma_2 \rangle
\end{array}
$$

**While 循环**：

$$
\begin{array}{c}
\langle e, \sigma \rangle \Downarrow \langle \mathbf{false}, \sigma_1 \rangle \\
\hline
\langle \mathbf{while}\ e\ s, \sigma \rangle \Downarrow \langle \mathbf{skip}, \sigma_1 \rangle
\end{array}
$$

$$
\begin{array}{c}
\langle e, \sigma \rangle \Downarrow \langle \mathbf{true}, \sigma_1 \rangle \quad
\langle s, \sigma_1 \rangle \Downarrow \langle \mathbf{skip}, \sigma_2 \rangle \quad
\langle \mathbf{while}\ e\ s, \sigma_2 \rangle \Downarrow \langle u, \sigma_3 \rangle \\
\hline
\langle \mathbf{while}\ e\ s, \sigma \rangle \Downarrow \langle u, \sigma_3 \rangle
\end{array}
$$

### 3.4.5 完整推导树示例

#### 示例 1：顺序与条件

**推导**：在初始空状态 $$\sigma_0 = \{\}$$ 下执行

```
x = 5;
y = x + 3;
if y > 0 then z = 1 else z = 0 end
```

即 $$s_1 = (x = 5) ; s_2$$，其中 $$s_2 = (y = x + 3) ; s_3$$，$$s_3 = \mathbf{if}\ (y > 0)\ (z = 1)\ (z = 0)$$。

初始状态：$$\sigma_0 = \{\}$$

**第一部分：求值 $$x = 5$$**

```
──────────────  (文字量规则)
⟨5, σ₀⟩ ⇓ ⟨5, σ₀⟩
──────────────────────  (赋值规则)
⟨x = 5, σ₀⟩ ⇓ ⟨skip, σ₁⟩
```
其中 $$\sigma_1 = \sigma_0\{x \mapsto 5\} = \{x \mapsto 5\}$$

**第二部分：求值 $$y = x + 3$$**

```
─────── (变量规则)
⟨x, σ₁⟩ ⇓ ⟨5, σ₁⟩         ─────────── (文字量规则)
                          ⟨3, σ₁⟩ ⇓ ⟨3, σ₁⟩
────────────────────────────────────────────── (加法规则)
⟨x + 3, σ₁⟩ ⇓ ⟨8, σ₁⟩
──────────────────────────────────────────────── (赋值规则)
⟨y = x + 3, σ₁⟩ ⇓ ⟨skip, σ₂⟩
```
其中 $$\sigma_2 = \sigma_1\{y \mapsto 8\} = \{x \mapsto 5, y \mapsto 8\}$$

**第三部分：求值条件**

```
─────── (变量规则)
⟨y, σ₂⟩ ⇓ ⟨8, σ₂⟩          ────────── (文字量规则)
                            ⟨0, σ₂⟩ ⇓ ⟨0, σ₂⟩
                    ──────────────────────────── (比较规则)
                    ⟨y > 0, σ₂⟩ ⇓ ⟨true, σ₂⟩
```

因为条件为 $$\mathbf{true}$$，走进 true 分支：

```
                                      ────────── (文字量规则)
                                      ⟨1, σ₂⟩ ⇓ ⟨1, σ₂⟩
                                  ────────────────────── (赋值规则)
⟨y > 0, σ₂⟩ ⇓ ⟨true, σ₂⟩        ⟨z = 1, σ₂⟩ ⇓ ⟨skip, σ₃⟩
────────────────────────────────────────────────────────── (if-true 规则)
⟨if y > 0 then z = 1 else z = 0, σ₂⟩ ⇓ ⟨skip, σ₃⟩
```
其中 $$\sigma_3 = \sigma_2\{z \mapsto 1\} = \{x \mapsto 5, y \mapsto 8, z \mapsto 1\}$$

**完整推导树**（合并）：

```
                     ... x+3 求值 ...
                    ──────────────────
⟨x=5, σ₀⟩ ⇓ ⟨*⟩    ⟨y=x+3, σ₁⟩ ⇓ ⟨*⟩           ... 条件求值 ...
───────────────────────────────────── (顺序)    ────────────────
⟨y=x+3, σ₁⟩ ⇓ ⟨skip, σ₂⟩                      ⟨if..., σ₂⟩ ⇓ ⟨skip, σ₃⟩
────────────────────────────────────────────────────────────────── (顺序)
⟨x=5; y=x+3; if y>0 then z=1 else z=0, σ₀⟩ ⇓ ⟨skip, σ₃⟩
```

最终状态：$$\sigma_3 = \{x \mapsto 5, y \mapsto 8, z \mapsto 1\}$$

#### 示例 2：While 循环的完整推导树

**推导**：在初始状态 $$\sigma_0 = \{x \mapsto 2\}$$ 下执行

```
while x > 0 do
  x = x - 1
end
```

我们将展开循环的每一次迭代。

**迭代 1**：

```
─────── (变量规则)
⟨x, σ₀⟩ ⇓ ⟨2, σ₀⟩           ────────── (文字量)
                             ⟨0, σ₀⟩ ⇓ ⟨0, σ₀⟩
                      ──────────────────────────── (比较规则)
                      ⟨x > 0, σ₀⟩ ⇓ ⟨true, σ₀⟩
```

条件为 $$\mathbf{true}$$，执行循环体：

```
... x > 0 为 true ...

⟨x, σ₀⟩ ⇓ ⟨2, σ₀⟩   ────────── (文字量)
                     ⟨1, σ₀⟩ ⇓ ⟨1, σ₀⟩
               ────────────────────────── (加法规则)
               ⟨x - 1, σ₀⟩ ⇓ ⟨1, σ₀⟩
               ────────────────────────── (赋值规则)
               ⟨x = x - 1, σ₀⟩ ⇓ ⟨skip, σ₁⟩
```

其中 $$\sigma_1 = \sigma_0\{x \mapsto 1\} = \{x \mapsto 1\}$$

**迭代 2**：从 $$\sigma_1 = \{x \mapsto 1\}$$ 开始

```
─────── (变量规则)
⟨x, σ₁⟩ ⇓ ⟨1, σ₁⟩           ────────── (文字量)
                             ⟨0, σ₁⟩ ⇓ ⟨0, σ₁⟩
                      ──────────────────────────── (比较规则)
                      ⟨x > 0, σ₁⟩ ⇓ ⟨true, σ₁⟩
```

条件仍为 $$\mathbf{true}$$，再次执行循环体：

```
⟨x, σ₁⟩ ⇓ ⟨1, σ₁⟩   ────────── (文字量)
                     ⟨1, σ₁⟩ ⇓ ⟨1, σ₁⟩
               ────────────────────────── (加法规则)
               ⟨x - 1, σ₁⟩ ⇓ ⟨0, σ₁⟩
               ────────────────────────── (赋值规则)
               ⟨x = x - 1, σ₁⟩ ⇓ ⟨skip, σ₂⟩
```

其中 $$\sigma_2 = \sigma_1\{x \mapsto 0\} = \{x \mapsto 0\}$$

**迭代 3**：从 $$\sigma_2 = \{x \mapsto 0\}$$ 开始

```
─────── (变量规则)
⟨x, σ₂⟩ ⇓ ⟨0, σ₂⟩           ────────── (文字量)
                             ⟨0, σ₂⟩ ⇓ ⟨0, σ₂⟩
                      ──────────────────────────── (比较规则)
                      ⟨x > 0, σ₂⟩ ⇓ ⟨false, σ₂⟩
```

条件为 $$\mathbf{false}$$，循环终止：

```
⟨x > 0, σ₂⟩ ⇓ ⟨false, σ₂⟩
─────────────────────────────── (while-false 规则)
⟨while x>0 do..., σ₂⟩ ⇓ ⟨skip, σ₂⟩
```

**完整推导树**：

```
... 迭代 1: ⟨x = x-1, σ₀⟩ ⇓ ⟨skip, σ₁⟩ ...

... 迭代 2: ⟨x = x-1, σ₁⟩ ⇓ ⟨skip, σ₂⟩ ...

... 迭代 3: ⟨x > 0, σ₂⟩ ⇓ ⟨false, σ₂⟩ ...
───────────────────────────────────────── while-false
⟨while x>0..., σ₂⟩ ⇓ ⟨skip, σ₂⟩
──────────────────────────────────────────────── while-true (迭代 2)
⟨while x>0..., σ₁⟩ ⇓ ⟨skip, σ₂⟩
──────────────────────────────────────────────── while-true (迭代 1)
⟨while x>0..., σ₀⟩ ⇓ ⟨skip, σ₂⟩
```

最终状态：$$\sigma_2 = \{x \mapsto 0\}$$（循环执行了 2 次，$$x$$ 从 2 递减到 0）。

#### 示例 3：嵌套 While 循环的推导

**推导**：在初始状态 $$\sigma_0 = \{x \mapsto 2, y \mapsto 0\}$$ 下执行

```
while x > 0 do
  y = y + x;
  x = x - 1
end
```

这相当于求和循环：$$0 + 2 + 1 = 3$$。

**迭代 1**：条件 $$x > 0$$ 为 $$\mathbf{true}$$（$$x=2$$）

```
⟨x, σ₀⟩ ⇓ ⟨2, σ₀⟩
⟨y, σ₀⟩ ⇓ ⟨0, σ₀⟩   ⟨x, σ₀⟩ ⇓ ⟨2, σ₀⟩
───────────────────────────── (加法)
⟨y + x, σ₀⟩ ⇓ ⟨2, σ₀⟩
──────────────────────────────── (赋值)
⟨y = y + x, σ₀⟩ ⇓ ⟨skip, σ₀{y↦2}⟩ = ⟨skip, σₐ⟩

其中 σₐ = {x↦2, y↦2}
```

然后执行 $$x = x - 1$$：

```
⟨x, σₐ⟩ ⇓ ⟨2, σₐ⟩   ⟨1, σₐ⟩ ⇓ ⟨1, σₐ⟩
───────────────────────────── (加法)
⟨x - 1, σₐ⟩ ⇓ ⟨1, σₐ⟩
──────────────────────────── (赋值)
⟨x = x - 1, σₐ⟩ ⇓ ⟨skip, σ₁⟩

其中 σ₁ = {x↦1, y↦2}
```

**迭代 2**：条件 $$x > 0$$ 为 $$\mathbf{true}$$（$$x=1$$）

```
⟨y, σ₁⟩ ⇓ ⟨2, σ₁⟩   ⟨x, σ₁⟩ ⇓ ⟨1, σ₁⟩
───────────────────────────── (加法)
⟨y + x, σ₁⟩ ⇓ ⟨3, σ₁⟩
──────────────────────────────── (赋值)
⟨y = y + x, σ₁⟩ ⇓ ⟨skip, σ_b⟩
其中 σ_b = {x↦1, y↦3}

然后 x = x - 1:
⟨x - 1, σ_b⟩ ⇓ ⟨0, σ_b⟩
⟨x = x - 1, σ_b⟩ ⇓ ⟨skip, σ₂⟩
其中 σ₂ = {x↦0, y↦3}
```

**迭代 3**：条件 $$x > 0$$ 为 $$\mathbf{false}$$（$$x=0$$），循环终止。

**完整推导树**：
```
... 迭代 1 的循环体 ...         ───────── while-true 第一次
⟨s₁;s₂, σ₀⟩ ⇓ ⟨skip, σ₁⟩      ⟨while x>0 ..., σ₁⟩ ⇓ ⟨skip, σ₂⟩
──────────────────────────────────────────────────────────────── while-true
⟨while x>0 do y=y+x; x=x-1, σ₀⟩ ⇓ ⟨skip, σ₂⟩
```

最终状态：$$\sigma_2 = \{x \mapsto 0, y \mapsto 3\}$$。$$\blacksquare$$

### 3.4.6 小步操作语义 (Small-Step Semantics) — 补充

与大步骤语义相对，**小步骤语义**通过一组归约规则描述程序执行过程中的每一步原子操作。

**小步骤归约关系**：$$\langle e, \sigma \rangle \to \langle e', \sigma' \rangle$$ 表示在当前状态 $$\sigma$$ 下，表达式 $$e$$ 一步归约到 $$e'$$ 且状态变为 $$\sigma'$$（对于语句）。

**表达式的小步语义**：

加法左操作数首先归约：
$$
\begin{array}{c}
\langle e_1, \sigma \rangle \to \langle e_1', \sigma' \rangle \\
\hline
\langle e_1 + e_2, \sigma \rangle \to \langle e_1' + e_2, \sigma' \rangle
\end{array}
$$

左操作数为值时，归约右操作数：
$$
\begin{array}{c}
\langle e_2, \sigma \rangle \to \langle e_2', \sigma' \rangle \\
\hline
\langle n + e_2, \sigma \rangle \to \langle n + e_2', \sigma' \rangle
\end{array}
$$

两者皆为值时，执行加法：
$$
\begin{array}{c}
\hline
\langle n_1 + n_2, \sigma \rangle \to \langle n, \sigma \rangle
\end{array}
$$
其中 $$n = n_1 + n_2$$。

**推导示例 (小步)**：执行 $$x = 1 + 2$$ 在 $$\sigma_0 = \{\}$$

```
⟨x = 1 + 2, σ₀⟩
→ ⟨x = 3, σ₀⟩                   [1+2 归约为 3]
→ ⟨skip, {x↦3}⟩                 [赋值执行]
```

**推导示例 (小步)**：执行 $$x = 1; y = x$$ 在 $$\sigma_0 = \{\}$$

```
⟨x = 1; y = x, σ₀⟩
→ ⟨skip; y = x, σ₁⟩            [x = 1 执行完毕, σ₁={x↦1}]
→ ⟨y = x, σ₁⟩                   [skip; 消去]
→ ⟨y = 1, σ₁⟩                   [x 求值为 1]
→ ⟨skip, σ₂⟩                    [赋值, σ₂={x↦1, y↦1}]
```

**小步与大步的比较**：

| 特性 | 大步骤语义 | 小步骤语义 |
|---|---|---|
| 单位 | 整个表达式到最终值 | 单步归约 |
| 不终止表达的表示 | 无法表示（无推导） | 可表示为无穷序列 |
| 交错 (interleaving) | 不支持 | 支持（并发语义的基础） |
| 规则复杂度 | 较简单 | 规则数量更多但更灵活 |
| 主要用途 | 编译器优化、类型保持 | 并发语义、归约系统 |

小步骤语义是定义并发语言的**进程代数 (process algebra)**（如 CCS、π-演算）和**归约语义 (reduction semantics)** 的基础。在现代编程语言的形式化中，小步骤语义能使描述更为精确——尤其在处理异常、非确定性和并发时，小步骤语义是唯一可行的选择。

### 3.4.7 Lambda 演算的操作语义

我们将对纯 lambda 演算定义操作语义。设 $$\Downarrow$$ 关系将表达式映射到其正规形式：

$$e_1 \Downarrow e_2$$ 表示表达式 $$e_1$$ 在正规序求值下归约到正规形式 $$e_2$$。

**规则**：

**规则 1：变量处于正规形式**：
$$
\begin{array}{c}
\hline
x \Downarrow x
\end{array}
$$

**规则 2：抽象处于正规形式**：
$$
\begin{array}{c}
\hline
\lambda x. e \Downarrow \lambda x. e
\end{array}
$$

**规则 3：应用——正规序 β-归约**：
$$
\begin{array}{c}
e_1 \Downarrow \lambda x. e \quad
e[x \mapsto e_2] \Downarrow v \quad
\mathbf{normal}(e_2) \\
\hline
e_1 e_2 \Downarrow v
\end{array}
$$

其中 $$\mathbf{normal}(e)$$ 为真当且仅当 $$e$$ 处于正规形式。

**辅助函数定义**：

$$\mathbf{normal}(x) = \mathbf{true}$$
$$\mathbf{normal}(\lambda x. e) = \mathbf{normal}(e)$$
$$\mathbf{normal}(e_1 e_2) = (\mathbf{normal}(e_1) \land \mathbf{normal}(e_2) \land (e_1 \text{ not an abstraction}))$$

最后一个条件确保 $$(\lambda x. e_1) e_2$$ 不被视为正规形式（虽然 $$\lambda x. e_1$$ 处于正规形式且 $$e_2$$ 也处于正规形式，但整体是一个 β-redex）。

**替换函数** $$\mathbf{subst}(e_1, x, e_2)$$：执行 $$e_1[x \mapsto e_2]$$，在必要时执行 α-替换以避免变量捕获。

**$$\mathbf{freevars}(e)$$**：返回表达式 $$e$$ 中自由变量的集合。

**$$\mathbf{boundvars}(e)$$**：返回表达式 $$e$$ 中绑定变量的集合。

**α-替换函数** $$\mathbf{alpha}(e, x, y)$$：将 $$e$$ 中绑定的 $$x$$ 重命名为 $$y$$。

#### 推导示例：$$(\lambda x. x)(\lambda y. y)$$

我们要证明 $$(\lambda x. x)(\lambda y. y) \Downarrow \lambda y. y$$。

**第一步：应用规则的前提**：

我们需要检查应用规则的三个前提：
1. $$e_1 \Downarrow \lambda x. e$$：即 $$(\lambda x. x) \Downarrow \lambda x. x$$
2. $$\mathbf{normal}(\lambda y. y)$$ 必须为真
3. $$e[x \mapsto e_2] \Downarrow v$$

**前提 1 的推导**：

```
x ⇓ x  (变量规则)
────────────── (抽象规则: 正规形式的前缀—函数体处于 nf)
λx. x ⇓ λx. x
```

实际上，根据规则 2，抽象自动处于正规形式：
```
────────────── (规则 2)
λx. x ⇓ λx. x
```

**前提 2 的检查**：

$$\mathbf{normal}(\lambda y. y)$$：
```
normal(λy. y) = normal(y) = normal(y)
```

由于 $$\mathbf{normal}(y) = \mathbf{true}$$（规则 1），有 $$\mathbf{normal}(\lambda y. y) = \mathbf{true}$$。$$\checkmark$$

**前提 3 的计算**：$$e[x \mapsto e_2] = x[x \mapsto (\lambda y. y)] = \lambda y. y$$

```
────────────── (规则 2)
λy. y ⇓ λy. y
```

**合并完整推导树**：

```
─────────── (规则2)    ────────────── (规则2)
λx.x ⇓ λx.x           λy.y ⇓ λy.y        normal(λy.y) = true
                          (x[x ↦ λy.y] = λy.y)
─────────────────────────────────────────────────────── (规则3: 应用)
(λx. x)(λy. y) ⇓ λy. y
```

结果验证：$$(\lambda x. x)$$ 是恒等函数，应用到 $$(\lambda y. y)$$ 上应该返回参数自身。$$\checkmark$$

## 3.5 形式类型系统 (Formal Type Systems)

类型系统在不运行程序的情况下对程序行为进行**静态近似**。如果程序通过类型检查，则保证在执行时不会出现某些类型的运行时错误——这就是 Robin Milner 的著名格言：

> Well-typed programs cannot "go wrong."（类型正确的程序不会"出错"）

### 3.5.1 类型判断 (Typing Judgments)

类型判断形式为：

$$\Gamma \vdash t : T$$

读作："在类型环境 $$\Gamma$$ 下，项 $$t$$ 具有类型 $$T$$"。

**类型环境 $$\Gamma$$** 是从变量名到类型的有限映射：

$$\Gamma = \{x_1: T_1, x_2: T_2, \dots, x_n: T_n\}$$

我们用 $$\Gamma, x: T$$ 表示在 $$\Gamma$$ 中扩展绑定 $$x: T$$（如果 $$x$$ 已在 $$\Gamma$$ 中则遮蔽旧绑定）。

### 3.5.2 Int/Bool 语言的类型规则

考虑一个简单的语言，包含整数、布尔值、加法、比较、逻辑与和条件。

**类型规则的公理 (Axioms)**：

**整数常量**：
$$
\begin{array}{c}
\hline
\Gamma \vdash n : \mathbf{Int}
\end{array}
$$

**布尔常量**：
$$
\begin{array}{c}
\hline
\Gamma \vdash \mathbf{true} : \mathbf{Bool}
\end{array}
\qquad
\begin{array}{c}
\hline
\Gamma \vdash \mathbf{false} : \mathbf{Bool}
\end{array}
$$

**变量**：
$$
\begin{array}{c}
x: T \in \Gamma \\
\hline
\Gamma \vdash x : T
\end{array}
$$

**加法**：
$$
\begin{array}{c}
\Gamma \vdash t_1 : \mathbf{Int} \quad \Gamma \vdash t_2 : \mathbf{Int} \\
\hline
\Gamma \vdash t_1 + t_2 : \mathbf{Int}
\end{array}
$$

**比较（大于）**：
$$
\begin{array}{c}
\Gamma \vdash t_1 : \mathbf{Int} \quad \Gamma \vdash t_2 : \mathbf{Int} \\
\hline
\Gamma \vdash t_1 > t_2 : \mathbf{Bool}
\end{array}
$$

**逻辑与**：
$$
\begin{array}{c}
\Gamma \vdash t_1 : \mathbf{Bool} \quad \Gamma \vdash t_2 : \mathbf{Bool} \\
\hline
\Gamma \vdash t_1 \,\&\, t_2 : \mathbf{Bool}
\end{array}
$$

**条件 (Conditional)**——初版（单一类型）：
$$
\begin{array}{c}
\Gamma \vdash c : \mathbf{Bool} \quad \Gamma \vdash t : T \quad \Gamma \vdash e : T \\
\hline
\Gamma \vdash \mathbf{if}\ c\ t\ e : T
\end{array}
$$

**条件**——通用版（带类型变量）：
$$
\begin{array}{c}
\Gamma \vdash c : \mathbf{Bool} \quad \Gamma \vdash t : T \quad \Gamma \vdash e : T \\
\hline
\Gamma \vdash \mathbf{if}\ c\ t\ e : T
\end{array}
$$

其中 $$T$$ 是一个类型变量，由 $$t$$ 和 $$e$$ 的具体类型实例化。

### 3.5.3 函数类型 (Function Types)

一旦引入 lambda 抽象，我们需要**函数类型**：

$$$$T_1 \rightarrow T_2$$$$

读作"从 $$T_1$$ 到 $$T_2$$ 的函数"。$$T_1$$ 是参数类型，$$T_2$$ 是返回类型。

**抽象规则 (Abstraction)**：

$$
\begin{array}{c}
\Gamma, x: T_1 \vdash e : T_2 \\
\hline
\Gamma \vdash \lambda x: T_1. e : T_1 \rightarrow T_2
\end{array}
$$

注意：参数必须带类型标注（这是简单类型 lambda 演算的要求，与无类型 lambda 演算不同）。

**应用规则 (Application)**：

$$
\begin{array}{c}
\Gamma \vdash e_1 : T_1 \rightarrow T_2 \quad \Gamma \vdash e_2 : T_1 \\
\hline
\Gamma \vdash e_1 e_2 : T_2
\end{array}
$$

### 3.5.4 完整类型推导树

#### 推导：$$\lambda f: \mathbf{Int} \rightarrow \mathbf{Int}. \lambda x: \mathbf{Int}. f (f x)$$

我们要推导类型：$$(\mathbf{Int} \rightarrow \mathbf{Int}) \rightarrow \mathbf{Int} \rightarrow \mathbf{Int}$$

**步骤 1：推导最内层的 $$f x$$**

设 $$\Gamma_0 = \{f: \mathbf{Int} \rightarrow \mathbf{Int}, x: \mathbf{Int}\}$$

```
f: Int→Int ∈ Γ₀            x: Int ∈ Γ₀
────────────── (变量)      ──────────── (变量)
Γ₀ ⊢ f : Int→Int          Γ₀ ⊢ x : Int
───────────────────────────────────────── (应用)
Γ₀ ⊢ f x : Int
```

**步骤 2：推导 $$f (f x)$$**

```
f: Int→Int ∈ Γ₀            ... 上面的 f x 推导 ...
────────────── (变量)      ─────────────────
Γ₀ ⊢ f : Int→Int          Γ₀ ⊢ f x : Int
───────────────────────────────────────────── (应用)
Γ₀ ⊢ f (f x) : Int
```

**步骤 3：抽象化 $$x$$**

```
Γ₀ = {f: Int→Int, x: Int} ⊢ f(f x) : Int
────────────────────────────────────────────── (抽象)
{f: Int→Int} ⊢ λx:Int. f(f x) : Int → Int
```

**步骤 4：抽象化 $$f$$**

```
{f: Int→Int} ⊢ λx:Int. f(f x) : Int → Int
────────────────────────────────────────────────── (抽象)
{} ⊢ λf:Int→Int. λx:Int. f(f x) : (Int→Int) → Int → Int
```

**完整推导树**（压缩格式）：

```
──────────(Var)  ──────────(Var)
f:Int→Int ⊢ f  x:Int ⊢ x
──────────────(App)      ──────────(Var)
f(fx) ⊢ ...  ───────────(Var)     f:Int→Int ⊢ f
──────────────(App)     ───────────────(App)
f:Int→Int, x:Int ⊢ f(fx):Int
──────────────────────────────(Abs)
f:Int→Int ⊢ λx:Int. f(fx) : Int→Int
──────────────────────────────────────(Abs)
⊢ λf:Int→Int. λx:Int. f(fx) : (Int→Int)→Int→Int
```

因此类型为 $$(\mathbf{Int} \rightarrow \mathbf{Int}) \rightarrow \mathbf{Int} \rightarrow \mathbf{Int}$$，这表示一个将 $$\mathbf{Int} \rightarrow \mathbf{Int}$$ 函数变换为 $$\mathbf{Int} \rightarrow \mathbf{Int}$$ 函数的高阶函数。$$\blacksquare$$

### 3.5.5 子类型 (Subtyping)

子类型关系 $$\mathbf{S <: T}$$ 表示"$$\mathbf{S}$$ 是 $$\mathbf{T}$$ 的子类型"。子类型关系的关键性质（Motivation）：**在期望 $$\mathbf{T}$$ 类型的值的地方，始终可以使用 $$\mathbf{S}$$ 类型的值**。这称为**替代原则 (Substitution Principle)** 或 **Liskov 替换原则 (Liskov Substitution Principle)**。

#### 子类型关系的形式定义

子类型是类型上的偏序关系（preorder）：它是自反的且传递的。

**自反性 (Reflexivity)**：
$$
\begin{array}{c}
\hline
T <: T
\end{array}
$$

**传递性 (Transitivity)**：
$$
\begin{array}{c}
S <: U \quad U <: T \\
\hline
S <: T
\end{array}
$$

**基础类型的子类型关系**：

- $$\mathbf{Bool}$$ 和 $$\mathbf{Int}$$ 互不为子类型。
- $$\mathbf{Float}$$ 是某些系统中用于引入数值子类型的类型：在某些系统中 $$\mathbf{Int} <: \mathbf{Float}$$。

**子类型蕴含规则 (Subsumption Rule)**：

$$
\begin{array}{c}
\Gamma \vdash t : S \quad S <: T \\
\hline
\Gamma \vdash t : T
\end{array}
$$

这正好体现了替换原则：如果 $$t$$ 有类型 $$S$$ 且 $$S <: T$$，则 $$t$$ 也有类型 $$T$$。

#### 联合 (Join)

两个类型的**联合 (join)** 是它们的最小公共超类型。

**定义**：$$T_1 \sqcup T_2 = T$$ 当且仅当：
1. $$T_1 <: T$$ 且 $$T_2 <: T$$
2. 对任意 $$U$$，如果 $$T_1 <: U$$ 且 $$T_2 <: U$$，则 $$T <: U$$

**$$\mathbf{Int} \sqcup \mathbf{Float}$$ 的推导**：

假设 $$\mathbf{Int} <: \mathbf{Float}$$：

- 条件 1：$$\mathbf{Int} <: \mathbf{Float}$$（假设）且 $$\mathbf{Float} <: \mathbf{Float}$$（自反性）$$\checkmark$$
- 条件 2：对任意 $$U$$，如果 $$\mathbf{Int} <: U$$ 且 $$\mathbf{Float} <: U$$，则 $$\mathbf{Float} <: U$$（自反性加传递性）$$\checkmark$$

因此 $$\mathbf{Int} \sqcup \mathbf{Float} = \mathbf{Float}$$。

**$$\mathbf{Int} \sqcup \mathbf{Bool}$$ 不存在**：

假设存在 $$T = \mathbf{Int} \sqcup \mathbf{Bool}$$。

- 条件 1 要求 $$\mathbf{Int} <: T$$ 且 $$\mathbf{Bool} <: T$$
- 但 $$\mathbf{Int}$$ 和 $$\mathbf{Bool}$$ 没有公共超类型（在简单类型系统中），二者代表完全不同的基础类型

因此 $$\mathbf{Int} \sqcup \mathbf{Bool}$$ 不存在（或等于 $$\top$$，如果系统有 $$\top$$ 类型）。

#### Top 类型

$$\top$$ 简称"顶类型"，是所有类型的超级：

$$
\begin{array}{c}
\hline
T <: \top
\end{array}
$$

如果系统有 $$\top$$：
- $$\mathbf{Int} \sqcup \mathbf{Bool} = \top$$
- $$\top \sqcup T = \top$$
- $$\top$$ 类似于 Java 的 `Object` 或 Scala 的 `Any`

#### 扩展的条件规则 (条件 + 联合)

之前我们要求条件的两个分支具有**相同类型**。有了子类型和联合，我们可以支持不同分支类型：

$$
\begin{array}{c}
\Gamma \vdash c : \mathbf{Bool} \quad
\Gamma \vdash t : T_1 \quad
\Gamma \vdash e : T_2 \\
\hline
\Gamma \vdash \mathbf{if}\ c\ t\ e : T_1 \sqcup T_2
\end{array}
$$

**推导：$$\mathbf{if\ true\ then\ 1\ else\ 2} : \mathbf{Int}$$**

```
───────(T)  ──────(Int)  ──────(Int)
true:Bool    1:Int      2:Int
                                 Int <: Int, Int <: Int
                                 Int ⊔ Int = Int
───────────────────────────────────────────────── (条件-联合)
if true then 1 else 2 : Int
```

**推导：$$\mathbf{if\ true\ then\ 1\ else\ 3.14} : \mathbf{Float}$$**

假设 $$\mathbf{Int} <: \mathbf{Float}$$ 且系统中有 $$\mathbf{Float}$$ 类型：

```
───────(T)  ──────(Int)  ──────(Float)
true:Bool    1:Int      3.14:Float
                                 1 有类型 Int, Int <: Float
                                 故 1 也有类型 Float
                                 Int ⊔ Float = Float
─────────────────────────────────────────────────── (条件-联合)
if true then 1 else 3.14 : Float
```

### 3.5.6 函数子类型 (Function Subtyping)

函数的子类型规则是类型系统中最微妙、最重要的部分之一——它引入了**协变/逆变**的概念。

**函数子类型规则**：

$$
\begin{array}{c}
T_1 <: S_1 \quad S_2 <: T_2 \\
\hline
S_1 \rightarrow S_2 <: T_1 \rightarrow T_2
\end{array}
$$

**核心规则**：
- **参数类型是逆变的 (contravariant)**：$$T_1 <: S_1$$（方向翻转！）
- **返回类型是协变的 (covariant)**：$$S_2 <: T_2$$（方向保留）

**直觉解释**：函数类型 $$F = S_1 \rightarrow S_2$$ 是 $$G = T_1 \rightarrow T_2$$ 的子类型，意味着任何接受 $$F$$ 的地方都可以接受 $$G$$。而 $$G$$ 能"放进" $$F$$ 的位置当且仅当：
1. $$G$$ 的参数更宽松（接受更多类型）：$$T_1$$ 是 $$S_1$$ 的**超类型**（$$S_1 <: T_1$$ 意味着要更精确的东西可以接受更粗略的东西——但从规则来看是 $$T_1 <: S_1$$，意味着 $$T_1$$ 是 $$S_1$$ 的子类型，即 $$T_1$$ **更精确** / 更"小"）
2. $$G$$ 的返回值更精确（返回更小类型）：$$S_2 <: T_2$$（$$S_2$$ 返回类型是 $$T_2$$ 的子类型）

**让我们用替代原则来理解**：我们希望在期望 $$S_1 \rightarrow S_2$$ 的地方使用 $$T_1 \rightarrow T_2$$。

- 调用者会传入一个 $$S_1$$ 类型的值。但 $$T_1 \rightarrow T_2$$ 期望的是 $$T_1$$。由于 $$T_1 <: S_1$$（来自 $$S_1 \rightarrow S_2 <: T_1 \rightarrow T_2$$ 推导所需的 $$T_1 <: S_1$$），实际上是 $$T_1$$ 是 $$S_1$$ 的子类型——这意味着 $$T_1$$ 的值集合是 $$S_1$$ 的子集。如果 $$S_1$$ 是更"大"的类型，那么 $$T_1$$ 可以接受 $$S_1$$ 类型的所有值（因为 $$S_1$$ 蕴含了 $$T_1$$ 的所有可能值？不...

**更精确的直观**：设 $$S_1 = \mathbf{Float}$$，$$T_1 = \mathbf{Int}$$（其他部分省略）。假设 $$S_1 \rightarrow S_2$$ 是一个"接受 Float 并返回某物"的函数。如果 $$T_1 \rightarrow T_2$$ 是"接受 Int 并返回某物"的函数，且我们知道 $$\mathbf{Int} <: \mathbf{Float}$$，那么 $$T_1$$（接受 Int）比 $$S_1$$（接受 Float）更**具体**/更**专业**。调用者传给 $$S_1 \rightarrow S_2$$ 一个 Float 值——Int 是 Float，所以没问题。调用者传给 $$T_1 \rightarrow T_2$$ 一个 Int——但 Int 的值总是可以传到 Float 的参数位置，所以 $$T_1 \rightarrow T_2$$ 类型的函数确实期望的是 Int 值，而调用者传入 Float 时可能导致类型不匹配...

实际上让我们用一个经典例子来验证。

#### 推导示例：$$(\mathbf{Float} \rightarrow \mathbf{Int}) <: (\mathbf{Int} \rightarrow \mathbf{Float})$$ 是否成立？

假设 $$\mathbf{Int} <: \mathbf{Float}$$。我们有：

```
S₁ = Float    T₁ = Int
S₂ = Int      T₂ = Float
```

要验证 $$S_1 \rightarrow S_2 <: T_1 \rightarrow T_2$$，需验证：
1. $$T_1 <: S_1$$，即 $$\mathbf{Int} <: \mathbf{Float}$$ — 成立
2. $$S_2 <: T_2$$，即 $$\mathbf{Int} <: \mathbf{Float}$$ — 成立

因此 $$(\mathbf{Float} \rightarrow \mathbf{Int}) <: (\mathbf{Int} \rightarrow \mathbf{Float})$$ $$\checkmark$$

直觉：$$\mathbf{Float} \rightarrow \mathbf{Int}$$ 比 $$\mathbf{Int} \rightarrow \mathbf{Float}$$ 更特殊（接受更灵活的参数 Float，返回更精确的结果 Int）。

#### 反例：$$(\mathbf{Int} \rightarrow \mathbf{Float}) <: (\mathbf{Float} \rightarrow \mathbf{Float})$$ 是否成立？

验证函数子类型规则：

```
S₁ = Int       T₁ = Float
S₂ = Float     T₂ = Float
```

需验证：
1. $$T_1 <: S_1$$，即 $$\mathbf{Float} <: \mathbf{Int}$$ — **不成立**（$$\mathbf{Float}$$ 不是 $$\mathbf{Int}$$ 的子类型）
2. $$S_2 <: T_2$$，即 $$\mathbf{Float} <: \mathbf{Float}$$ — 成立（自反性）

因此 $$(\mathbf{Int} \rightarrow \mathbf{Float}) \nless: (\mathbf{Float} \rightarrow \mathbf{Float})$$ $$\blacksquare$$

**为什么失败**：如果这成立，我们就可以在期望 $$\mathbf{Float} \rightarrow \mathbf{Float}$$ 的地方使用 $$\mathbf{Int} \rightarrow \mathbf{Float}$$。调用者传入 $$\mathbf{Float}$$ 值，但函数体期望 $$\mathbf{Int}$$ 值——类型不匹配！这就是**逆变**的力量：参数类型的方向必须翻转，否则会破坏类型安全。

#### 推导：用子类型蕴含规则证明类型判断

**问题**：推导 $$\lambda f: \mathbf{Float} \rightarrow \mathbf{Int}. f\ 3 : (\mathbf{Float} \rightarrow \mathbf{Int}) \rightarrow \mathbf{Int}$$

设 $$\Gamma = \{f: \mathbf{Float} \rightarrow \mathbf{Int}\}$$

```
步骤 1: 3 : Int (常量)
步骤 2: Int <: Float (假设)
步骤 3: 3 : Float (步骤 1 + 步骤 2 + 子类型蕴含)
步骤 4: f : Float → Int (假设)
步骤 5: f 3 : Int (步骤 4 + 步骤 3 + 应用规则)
步骤 6: λf:Float→Int. f 3 : (Float→Int) → Int (抽象规则)
```

完整推导树：

```
───────────── (Int)
3 : Int         Int <: Float
─────────────(Sub)        ───────────── (Var)
3 : Float                f:Float→Int ⊢ f : Float→Int
────────────────────────────────────────────────────(App)
f:Float→Int ⊢ f 3 : Int
────────────────────────────────────────(Abs)
⊢ λf:Float→Int. f 3 : (Float→Int) → Int
```

### 3.5.7 子类型应用的综合示例

**问题**：推导 $$\lambda f: \mathbf{Int} \rightarrow \mathbf{Float}. \lambda g: \mathbf{Float} \rightarrow \mathbf{Int}. g (f\ 5)$$ 的类型。

设 $$\Gamma = \{f: \mathbf{Int} \rightarrow \mathbf{Float}, g: \mathbf{Float} \rightarrow \mathbf{Int}\}$$

```
步骤 1: 5 : Int (常量)
步骤 2: f : Int → Float (假设)
步骤 3: f 5 : Float (步骤 1 + 步骤 2 + 应用规则)
步骤 4: g : Float → Int (假设)
步骤 5: g (f 5) : Int (步骤 3 + 步骤 4 + 应用规则)
```

需要抽象化 $$g$$ 和 $$f$$：

```
Γ ⊢ g (f 5) : Int
───────────────────────────────── (Abs)
f:Int→Float ⊢ λg:Float→Int. g(f 5) : (Float→Int) → Int
────────────────────────────────────────────────────────── (Abs)
⊢ λf:Int→Float. λg:Float→Int. g(f 5) : (Int→Float) → (Float→Int) → Int
```

完整类型：$$(\mathbf{Int} \rightarrow \mathbf{Float}) \rightarrow (\mathbf{Float} \rightarrow \mathbf{Int}) \rightarrow \mathbf{Int}$$

## 3.6 形式类型系统的可靠性 (Soundness)

类型系统的核心定理是**可靠性 (Soundness)**，包括两个引理：

**引理 1：进展 (Progress)** — 如果一个闭合项 $$t$$ 有类型 $$T$$，那么要么 $$t$$ 是一个值（已归约到值），要么 $$t$$ 可以进一步归约（$$\exists t'. t \to t'$$）。换句话说，类型正确的程序不会"卡住"。

**引理 2：保持 (Preservation) / 主类型 (Subject Reduction)** — 如果 $$\Gamma \vdash t : T$$ 且 $$t \to t'$$，则 $$\Gamma \vdash t' : T$$。换句话说，归约保持类型。

结合这两个引理，可得**可靠性**：如果 $$\vdash t : T$$ 且 $$t \to^* t'$$ 且 $$t'$$ 无法继续归约，则 $$t'$$ 是一个类型为 $$T$$ 的值。类型正确的程序永远不会进入"出错"状态。

### 3.6.1 进展和保持的形式证明概要 (Proof Sketch)

**进展的证明**（归纳于类型推导）：
- 如果最后的规则是常量规则：$$n$$ 或 $$b$$ 是值（$$\checkmark$$ 进展）
- 如果最后的规则是变量规则：$$x$$ 是值（$$\checkmark$$ 进展）——但注意对于闭合程序，变量规则不会被使用
- 如果最后的规则是加法规则：$$\Gamma \vdash e_1 : \mathbf{Int}$$ 且 $$\Gamma \vdash e_2 : \mathbf{Int}$$。由归纳假设，$$e_1$$ 要么是值要么可以归约。如果是值，再由归约假设判断 $$e_2$$。
- 如果最后的规则是应用规则：$$\Gamma \vdash e_1 : T_1 \rightarrow T_2$$ 且 $$\Gamma \vdash e_2 : T_1$$。由归纳假设，$$e_1$$ 要么是值要么可以归约。如果 $$e_1$$ 是值（必为抽象 $$\lambda x: T_1. e$$），则由归约假设判断 $$e_2$$。如果 $$e_2$$ 是值，则 $$(\lambda x: T_1. e) e_2$$ 是一个 redex——可归约。（$$\checkmark$$ 进展）

**保持的证明**（归纳于类型推导）：
- 关键情形是 β-归约：$$(\lambda x: T_1. e) v \to e[x \mapsto v]$$。
  - 前提：$$\Gamma \vdash (\lambda x: T_1. e) : T_1 \rightarrow T_2$$ 且 $$\Gamma \vdash v : T_1$$
  - 抽象规则给出：$$\Gamma, x: T_1 \vdash e : T_2$$
  - 需要证明：$$\Gamma \vdash e[x \mapsto v] : T_2$$
  - 这需要**替换引理 (Substitution Lemma)**：如果 $$\Gamma, x: S \vdash e : T$$ 且 $$\Gamma \vdash v : S$$，则 $$\Gamma \vdash e[x \mapsto v] : T$$

替换引理的证明又需对类型推导进行归纳。这是类型系统可靠性的核心。

### 3.6.2 积类型与和类型 (Product and Sum Types)

纯 lambda 演算的简单类型系统可以扩展以包含结构化类型。

**积类型 (Product Types / Pairs)**：

语法扩展：$$T_1 \times T_2$$。

引入规则：
$$
\begin{array}{c}
\Gamma \vdash e_1 : T_1 \quad \Gamma \vdash e_2 : T_2 \\
\hline
\Gamma \vdash (e_1, e_2) : T_1 \times T_2
\end{array}
$$

消去规则（投影）：
$$
\begin{array}{c}
\Gamma \vdash e : T_1 \times T_2 \\
\hline
\Gamma \vdash \mathbf{fst}\ e : T_1
\end{array}
\quad
\begin{array}{c}
\Gamma \vdash e : T_1 \times T_2 \\
\hline
\Gamma \vdash \mathbf{snd}\ e : T_2
\end{array}
$$

**积类型的子类型**：

$$
\begin{array}{c}
S_1 <: T_1 \quad S_2 <: T_2 \\
\hline
S_1 \times S_2 <: T_1 \times T_2
\end{array}
$$

积类型在**两个分量上都是协变的**。

**推导**：$$\mathbf{Int} \times \mathbf{Int} <: \mathbf{Float} \times \mathbf{Float}$$

假设 $$\mathbf{Int} <: \mathbf{Float}$$：
```
Int <: Float      Int <: Float
─────────────────────────────── (积子类型)
Int × Int <: Float × Float
     ✓
```

**和类型 (Sum Types / Tagged Unions)**：

语法：$$T_1 + T_2$$。

引入规则（左注入和右注入）：
$$
\begin{array}{c}
\Gamma \vdash e : T_1 \\
\hline
\Gamma \vdash \mathbf{inl}\ e : T_1 + T_2
\end{array}
\quad
\begin{array}{c}
\Gamma \vdash e : T_2 \\
\hline
\Gamma \vdash \mathbf{inr}\ e : T_1 + T_2
\end{array}
$$

消去规则（case 分析）：
$$
\begin{array}{c}
\Gamma \vdash e : T_1 + T_2 \quad
\Gamma, x: T_1 \vdash e_1 : T \quad
\Gamma, y: T_2 \vdash e_2 : T \\
\hline
\Gamma \vdash \mathbf{case}\ e\ \mathbf{of}\ \mathbf{inl}\ x \mapsto e_1\ |\ \mathbf{inr}\ y \mapsto e_2 : T
\end{array}
$$

**和类型的子类型**：

$$
\begin{array}{c}
S_1 <: T_1 \quad S_2 <: T_2 \\
\hline
S_1 + S_2 <: T_1 + T_2
\end{array}
$$

和类型在**两个分量上也都是协变的**。

### 3.6.3 参数多态 (Parametric Polymorphism) — 简介

简单类型系统的主要局限是**缺乏复用性**：每个函数必须针对特定类型编写。例如，恒等函数需要为每种类型各写一个：

```
λx:Int. x : Int → Int
λx:Bool. x : Bool → Bool
```

**参数多态 (parametric polymorphism)** 引入了**类型变量**和**全称量化**，允许一个函数适用于所有类型：

$$$$\mathbf{id} = \Lambda \alpha. \lambda x: \alpha. x : \forall \alpha. \alpha \rightarrow \alpha$$$$

这里 $$\Lambda \alpha. e$$ 是**类型抽象 (type abstraction)**，$$\forall \alpha. T$$ 是**全称类型 (universal type)**。

**多态恒等函数的类型规则**：

$$
\begin{array}{c}
\alpha \ \text{type}, x: \alpha \vdash x : \alpha \\
\hline
\alpha \ \text{type} \vdash \lambda x: \alpha. x : \alpha \rightarrow \alpha \\
\hline
\vdash \Lambda \alpha. \lambda x: \alpha. x : \forall \alpha. \alpha \rightarrow \alpha
\end{array}
$$

**多态应用（类型实例化）**：

$$\mathbf{id} [\mathbf{Int}]\ 42$$  将 $$\forall \alpha. \alpha \rightarrow \alpha$$ 实例化为 $$\mathbf{Int} \rightarrow \mathbf{Int}$$。

系统 F（Girard-Reynolds）是融合了参数多态的 lambda 演算的形式化，它是现代函数式语言类型系统的理论基础。其类型推导规则在后续章节中展开讨论。关于 System F 的可靠性同样是进展+保持的形式，这也是所有类型系统可靠性的共同范式。

### 3.6.4 带子类型的类型推导完整示例

**问题**：给出以下多态程序的类型推导

```
let twice = λf:Int→Int. λx:Int. f(f x) in
twice (λy:Float. y) 3
```

（简化处理：将 let 展开且忽略多态，使用 Int <: Float。）

**推导**：

首先，``twice'' 的类型是 $$(\mathbf{Int} \rightarrow \mathbf{Int}) \rightarrow \mathbf{Int} \rightarrow \mathbf{Int}$$（见之前的推导）。

但参数 $$\lambda y: \mathbf{Float}. y$$ 的类型是 $$\mathbf{Float} \rightarrow \mathbf{Float}$$。

我们需要检查 $$(\mathbf{Float} \rightarrow \mathbf{Float}) <: (\mathbf{Int} \rightarrow \mathbf{Int})$$。

按函数子类型规则：
```
需要: Int <: Float （参数逆变） 且 Float <: Int （返回协变）
```

$$\mathbf{Int} <: \mathbf{Float}$$ 成立（假设），但 $$\mathbf{Float} <: \mathbf{Int}$$ **不成立**。

因此**类型检查失败**！这实际上是合理的——如果 $$\mathbf{twice}$$ 期望一个 $$\mathbf{Int} \rightarrow \mathbf{Int}$$ 函数，它可能将返回值作为 $$\mathbf{Int}$$ 使用。但 $$\lambda y: \mathbf{Float}. y$$ 返回的是 $$\mathbf{Float}$$，不能安全地当 $$\mathbf{Int}$$ 用。

这个反例展示了函数子类型检查如何保守地但正确地防范类型错误。$$\blacksquare$$

## 本章总结

本章系统构建了编程语言的理论基础：

1. **Lambda 演算**：一种只有变量、抽象和应用三种语法的图灵完备计算模型。α-变换保证了变量重命名无关性；β-归约实现了函数调用的形式化。正规序求值是最左最外归约策略，保证在正规形式存在时能够找到它。$$\Omega$$ 组合子展示了 lambda 演算的非终止能力。

2. **Church 编码**：在纯 lambda 演算中编码布尔值、数对和自然数。Church 数 $$n$$ 就是一个将函数 $$f$$ 应用到 $$x$$ 上 $$n$$ 次的函数。所有标准算术运算（加法、乘法、零测试）都可以通过纯 lambda 演算实现。

3. **递归与 Y 组合子**：不动点组合子 $$Y = \lambda f. (\lambda x. f (x x))(\lambda x. f (x x))$$ 满足 $$Y F =_β F (Y F)$$，使得在不支持显式递归的语言中实现递归。

4. **操作语义**：大步骤语义通过状态 $$\sigma$$ 和转移关系 $$\Downarrow$$ 严格定义了程序的执行过程。每个执行步骤对应一棵推导树。

5. **形式类型系统**：类型判断 $$\Gamma \vdash t : T$$ 静态地近似程序行为。子类型关系通过自反性、传递性和函数类型的逆变/协变规则构成。类型可靠性（进展 + 保持）是类型系统的核心保证。

6. **求值策略**：正规序、应用序和惰性求值代表了不同的实现策略选择，各自在终止性保证和性能效率之间取得不同平衡。正规序保证找到正规形式（如果存在），应用序通常更高效，惰性求值通过共享机制结合了两者优势。

7. **计算等价性**：Lambda 演算、图灵机、递归函数理论（Kleene 的一般递归函数）等模型在计算能力上是完全等价的。这一等价性由 Church-Turing 论题概括，是计算科学最深远的理论结果之一。停机问题的不可判定性进一步划定了计算的边界。

### 理论基础与编程实践的联系

理解本章的概念对日常编程实践有直接影响：

- **β-归约**对应函数调用和参数替换，是所有编程语言函数语义的基础。
- **Church 编码**的思想直接体现在高阶函数中——例如 JavaScript 的函数式编程模式、Python 的装饰器。
- **操作语义**为程序分析和编译器验证提供形式基础。
- **子类型和协变/逆变**在 Java 泛型（`? extends T`、`? super T`）、TypeScript 和 C# 的类型系统中具有直接应用。
- **不动点组合子**解释了递归在纯函数式语言中的实现方式。
- **类型可靠性**是现代类型安全的编程语言设计的基石。

从 Church 的 lambda 演算到今天的 Rust、Haskell、TypeScript，形式理论一直在推动编程语言的演进。掌握本章就是掌握编程语言设计的 DNA。
